🏗️ Guide de Création de Bâtiments pour l'Atelier

Version: 1.0 | Dernière mise à jour: 2026 | Pour: Unity 2022.3 LTS

Bienvenue, Créateur ! Merci de votre intérêt pour la création de contenu personnalisé ! Vos contributions aideront la communauté à prospérer ! Ce guide vous aidera à créer du contenu conforme aux spécifications et à le télécharger sur le Workshop Steam.

Exigences Techniques

Le téléchargement de contenu dépend de deux scripts : BuildingData et WorkshopUploadTool. BuildingData est utilisé pour remplir les données du bâtiment, et WorkshopUploadTool est utilisé pour le téléchargement. Vous avez également besoin de Steamworks.NET. Si vous devez mettre à jour Steamworks.NET, vous pouvez le télécharger depuis la page GitHub :

https://github.com/rlabrecque/Steamworks.NET/releases

Importez-le dans Unity, et attachez le script Steam Manager à un GameObject vide.

💡 Astuce: J'ai empaqueté les scripts requis et Steamworks.NET dans un fichier de package Unity. Vous pouvez utiliser ce fichier de package ou les télécharger séparément.

Spécifications du Modèle de Bâtiment

⚠️ Rappel des Composants Requis: Les prefabs de bâtiment doivent contenir les trois composants principaux suivants. L'absence de l'un d'entre eux entraînera des dysfonctionnements :
  • Mesh Filter : Affiche le maillage du modèle
  • Mesh Renderer : Rend le modèle (nécessite l'assignation d'un matériau)
  • Collider (Box Collider) : Utilisé pour la surbrillance au survol de la souris, la détection des clics et la détection de collision de placement

Formats de Ressources Supportés

  • Modèles : FBX
  • Textures : PNG
  • Matériaux : Unity URP Lit Shader

Limites de Taille de Fichier

  • AssetBundle de bâtiment unique : ≤ 50 Mo
  • Image de prévisualisation : ≤ 1 Mo
  • Taille totale de téléchargement : ≤ 100 Mo

Directives de Conception des Bâtiments

1. Spécifications de Taille

Concevez en fonction de la taille de grille occupée par le bâtiment. La base du bâtiment (fondation) doit mesurer au minimum 1 mètre de longueur et largeur :

Pour que l'occupation de grille du bâtiment soit correctement calculée, il est recommandé d'utiliser des carrés impairs pour la taille de base du bâtiment, tels que : 1×1, 3×3, 5×5. Les carrés pairs et les formes non carrées ne sont pas recommandés, tels que : 1×2, 2×2, 2×3, 4×4.

Hauteur : Aucune limite stricte, mais il est recommandé de ne pas descendre en dessous de 0,1 mètre

2. Style Visuel

Pour maintenir la cohérence du jeu, il est recommandé de :

  • Utiliser le style LowPoly
  • Assurer les proportions globales et les relations d'échelle du bâtiment (référence d'échelle par défaut du jeu : hauteur du personnage 0,1 mètre)
  • Utiliser des designs adaptés à la vue isométrique
  • Éviter les détails trop petits (difficiles à voir de loin)
  • Utiliser des contours clairs et un contraste de couleurs
  • Prendre en compte les détails sous différents angles

3. Recommandations d'Optimisation des Performances

  • Contrôler raisonnablement le nombre de polygones
  • Matériaux transparents (utiliser Alpha Test au lieu d'Alpha Blend)

Processus de Création

Étape 1 : Modélisation dans un Logiciel 3D

  1. Créer des modèles avec Blender, Maya, 3ds Max, etc.
  2. Assurer que l'origine du modèle est au centre du bas
    • ✅ Correct : Point central du bâtiment à l'origine du monde
    • ❌ Incorrect : Bâtiment décalé ou rotation non alignée
  3. Appliquer toutes les transformations (Scale, Rotation)
  4. Exporter au format FBX

Étape 2 : Importation dans Unity

  1. Faire glisser le FBX dans le projet Unity
  2. Définir la Position Transform du prefab de bâtiment sur (0, 0, 0)
  3. Assurer que la base du bâtiment est sur le plan Y=0
  4. Le bâtiment doit faire face à la direction positive de l'axe Z

Étape 3 : Définir les Matériaux et Textures

  1. Créer des matériaux (en utilisant URP Lit Shader)
  2. Ajuster les paramètres du matériau

Étape 4 : Ajouter les Composants Requis

Assurer que le prefab de bâtiment contient les trois composants requis suivants :

1. Mesh Filter et Mesh Renderer

  1. Sélectionner l'objet racine du bâtiment
  2. Confirmer que le composant Mesh Filter a été ajouté (sinon, cliquer sur Add Component → Mesh Filter)
  3. Confirmer que le composant Mesh Renderer a été ajouté
  4. Assigner des matériaux dans le Mesh Renderer (utiliser Standard ou URP Lit Shader)
⚠️ Important: Les bâtiments sans Mesh Renderer ne s'afficheront pas et ne déclencheront pas la surbrillance au survol de la souris !

2. Collider

  1. Sélectionner l'objet racine du bâtiment
  2. Ajouter un composant Box Collider (recommandé) ou Mesh Collider
  3. Ajuster la taille du Collider pour couvrir tout le bâtiment
  4. Assurer que le Collider ne dépasse pas la zone du bâtiment, car cela provoquerait des collisions avec d'autres bâtiments
⚠️ Important: Le Collider est une condition nécessaire pour la surbrillance au survol de la souris et la détection des clics ! Les bâtiments sans Collider ne peuvent pas être sélectionnés ou interagis par les joueurs.

Étape 5 : Créer le Prefab

  1. Faire glisser le bâtiment dans la fenêtre Project pour créer un Prefab
  2. Convention de nommage : {BuildingName}_Prefab
  3. Tester que le prefab peut être instancié normalement

Étape 6 : Créer BuildingData

  1. Clic droit → Create → City Builder → Building Data
  2. Remplir les champs suivants :
Champ Description Exemple
Building Name Nom d'affichage du bâtiment, peut utiliser un nom ou un numéro Ferriswheel, SSmbl2
Building ID Identifiant unique, recommandé : nom de style + numéro, ou numéro pur SunsetCity_01, SSmbl2
Style ⭐ Détermine sous quel onglet de style le bâtiment apparaît dans le menu de construction du jeu, recommandé : nom du développeur, nom du package ou nom du thème. Les bâtiments de la même série doivent avoir la même valeur Lyon'sPack, SunsetCity
Category ⭐ Détermine sous quelle colonne de catégorie il apparaît sous ce style, peut utiliser les catégories de jeu existantes pour la compatibilité, ou remplir selon la classification de votre bâtiment Large, Residential
Grid Size Taille d'occupation de grille du bâtiment 3x3
Building Prefab Faire glisser votre prefab de bâtiment —

Référence de catégorie intégrée au jeu :

Valeur de catégorie Description
LargeGrand
MediumMoyen
SmallPetit
⚠️ Les champs Style et Category sont très importants : Ils seront empaquetés dans metadata.json (écrit automatiquement par l'outil de téléchargement). Lorsque le jeu charge votre mod, il détermine la position du bâtiment dans le menu en fonction de ces deux champs. Veuillez vous assurer de les remplir, sinon le bâtiment pourrait ne pas s'afficher dans la bonne catégorie.
  1. Créer une icône de bâtiment (recommandé 128×128 PNG) et la faire glisser dans le champ Icon

Étape 7 : Tester le Bâtiment

  1. Tester le bâtiment dans le jeu :
    • Peut-il être placé correctement
    • La rotation est-elle normale
    • La détection de collision est-elle correcte
    • Les effets visuels sont-ils satisfaisants
  2. Ajuster jusqu'à satisfaction

Étape 8 : Préparation au Téléchargement

  • Créer une image de prévisualisation (recommandé 600x600)
  • Confirmer que Style et Category dans BuildingData sont correctement remplis (l'outil de téléchargement les lit automatiquement)

Liste de vérification avant téléchargement :

  • Le nombre de triangles du modèle est dans les limites
  • La résolution de texture est raisonnable
  • Contient le composant Mesh Filter
  • Contient le composant Mesh Renderer (matériau assigné)
  • Contient le composant Collider
  • Le prefab peut être instancié normalement
  • Le bâtiment est aligné à la grille
  • Pas de matériaux ou textures manquants
  • Tous les champs BuildingData sont remplis
  • L'image de prévisualisation est claire
  • Test en jeu réussi

Étape 9 : Télécharger sur l'Atelier

⚠️ Le téléchargement se fait en deux étapes, car l'empaquetage AssetBundle et le téléchargement Steam ont des exigences différentes pour l'état de l'éditeur et doivent être opérés séparément.

Étape 1 : Empaqueter les Ressources (Mode Non-Play)

  1. Assurer d'être actuellement en Mode Non-Play (éditeur non en cours d'exécution)
  2. Ouvrir City Builder → Workshop Upload Tool
  3. Sélectionner le type de mod : Building
  4. Faire glisser BuildingData (l'outil lit automatiquement Style et Category, peut être modifié manuellement)
  5. Faire glisser le prefab de bâtiment
  6. Remplir les informations de l'atelier (principalement le titre, la description peut être éditée dans l'Atelier)
  7. Cliquer sur "Étape 1 : Empaqueter les ressources comme AssetBundle"
  8. Lorsque vous voyez le message "Empaquetage terminé ! Veuillez entrer en Mode Play et cliquer sur l'Étape 2 pour télécharger", c'est réussi
💡 Astuce: Le chemin d'empaquetage sera automatiquement sauvegardé et ne sera pas perdu après être entré en Mode Play, pas besoin de remplir à nouveau les informations.

Étape 2 : Télécharger sur Steam (Mode Play)

  1. Cliquer sur le bouton Play de l'éditeur Unity et attendre l'initialisation de SteamManager
  2. Dans la fenêtre de l'outil, confirmer que Steam s'est initialisé (pas d'avertissements)
  3. Cliquer sur "Étape 2 : Télécharger sur Steam Workshop"
  4. Attendre que le téléchargement soit terminé et noter l'ID d'élément Workshop affiché (pour les mises à jour ultérieures)
💡 Astuce: L'outil de téléchargement génère automatiquement metadata.json et l'empaquete dans le répertoire de contenu AssetBundle, qui contient les informations Style et Category. Après que les joueurs se soient abonnés, le jeu l'affichera automatiquement dans la bonne catégorie.

Étape 10 : Publier et Maintenir

  1. Compléter les informations sur la page Steam Workshop
  2. Ajouter plus de captures d'écran

Notes sur les Matériaux de Ciel et de Sol

  • Les shaders de ciel et de sol doivent utiliser le pipeline URP
  • Le shader de sol devrait utiliser des normales procédurales pour gérer les ajustements dynamiques de taille de zone de grille

Erreurs Courantes et Solutions

Erreur 0 : "Building AssetBundles while in play mode is not allowed" lors de l'empaquetage

Cause : Le bouton d'empaquetage a été cliqué en Mode Play, mais l'empaquetage AssetBundle ne peut être exécuté qu'en Mode Non-Play

Solution :

  1. Quitter le Mode Play (cliquer sur le bouton d'arrêt)
  2. Re-cliquer sur "Étape 1 : Empaqueter les ressources comme AssetBundle" dans l'outil
  3. Entrer en Mode Play pour télécharger après l'empaquetage terminé

Erreur 1 : Le bâtiment apparaît rose

Cause : Matériau ou shader manquant

Solution :

  1. Assurer que toutes les textures sont correctement assignées
  2. Vérifier si les matériaux sont inclus dans l'AssetBundle

Erreur 2 : Le bâtiment ne peut pas être placé

Cause : Composants requis manquants (Mesh Filter/Mesh Renderer/Collider) ou paramètre de taille de grille incorrect

Solution :

  1. Confirmer que le prefab de bâtiment contient les composants Mesh Filter, Mesh Renderer et Collider
  2. Vérifier si des matériaux sont assignés au Mesh Renderer
  3. Ajouter Box Collider à l'objet racine du bâtiment
  4. Vérifier le paramètre Grid Size de BuildingData
  5. Confirmer les dimensions du bâtiment

Erreur 3 : Position du bâtiment décalée

Cause : Paramètres Transform du prefab incorrects

Solution :

  1. Définir la Position du prefab sur (0, 0, 0)
  2. Assurer que la base du bâtiment est sur le plan Y=0
  3. Vérifier la position du Pivot Point du modèle

Erreur 4 : Bâtiment mal aligné après rotation

Cause : Le point central du bâtiment n'est pas au centre de la grille

Solution :

  1. Ajuster l'origine du modèle dans le logiciel 3D
  2. Ou ajuster la position des objets enfants dans Unity
  3. Assurer que le bâtiment tourne autour du point central

Erreur 5 : Fichier trop volumineux pour être téléchargé

Cause : Résolution de texture trop élevée ou modèle trop complexe

Solution :

  1. Réduire la résolution de texture (ex. de 4K à 2K)
  2. Optimiser le modèle pour réduire le nombre de polygones
  3. Compresser les textures (utiliser le format DXT5 ou BC7)
  4. Supprimer les détails inutiles

Techniques Avancées

Ajouter une Animation

Si votre bâtiment contient une animation (comme un moulin à vent rotatif) :

  1. Créer un contrôleur d'animation et faire glisser l'animation dedans
  2. Ajouter un composant Animator au prefab
  3. Faire glisser le contrôleur d'animation dans le composant Animator

Utiliser le Système LOD

Ajouter un script de suppression de détails pour les grands bâtiments :

  1. Ajouter le script BuildingDetailCulling
  2. Assigner différents niveaux LOD au modèle
  3. Définir différentes distances de suppression pour chaque niveau

Effets de Matériau Personnalisés

Utiliser Shader Graph pour créer des effets spéciaux :

  1. Créer un asset Shader Graph
  2. Concevoir des effets personnalisés (comme des fenêtres lumineuses)
  3. Créer un matériau utilisant ce Shader
  4. Assurer que le Shader est compatible avec la plateforme cible
