🏗️ Workshop Gebäude-Erstellungsleitfaden

Version: 1.0 | Letzte Aktualisierung: 2026 | Für: Unity 2022.3 LTS

Willkommen, Creator! Vielen Dank für Ihr Interesse an der Erstellung von benutzerdefinierten Inhalten! Ihr Beitrag wird der Community zum Gedeihen verhelfen! Dieser Leitfaden hilft Ihnen, Inhalte zu erstellen, die den Spezifikationen entsprechen, und sie in die Steam Workshop hochzuladen.

Technische Anforderungen

Das Hochladen von Inhalten erfordert zwei Skripte: BuildingData und WorkshopUploadTool. BuildingData wird zum Ausfüllen der Gebäudedaten verwendet, und WorkshopUploadTool wird zum Hochladen verwendet. Sie benötigen auch Steamworks.NET und die steam_appid.txt-Datei. Die steam_appid.txt-Datei sollte im Unity-Projekt-Stammverzeichnis mit der Spiel-ID: 4490290 platziert werden. Wenn Sie Steamworks.NET aktualisieren müssen, können Sie es von der GitHub-Seite herunterladen:

https://github.com/rlabrecque/Steamworks.NET/releases

Importieren Sie es in Unity, und hängen Sie das Steam Manager-Skript an ein leeres GameObject an.

💡 Tipp: Ich habe die erforderlichen Skripte und Steamworks.NET in eine Unity-Paketdatei gepackt. Sie können diese Paketdatei verwenden oder sie separat herunterladen.

Gebäudemodell-Spezifikationen

⚠️ Erforderliche Komponenten Erinnerung: Gebäude-Prefabs müssen die folgenden drei Kernkomponenten enthalten. Das Fehlen einer davon führt zu Funktionsstörungen:
  • Mesh Filter: Zeigt das Modell-Mesh an
  • Mesh Renderer: Rendert das Modell (erfordert Materialzuweisung)
  • Collider (Box Collider): Wird für Maus-Hover-Hervorhebung, Klick-Erkennung und Platzierungs-Kollisionserkennung verwendet

Unterstützte Ressourcenformate

  • Modelle: FBX
  • Texturen: PNG
  • Materialien: Unity URP Lit Shader

Dateigrößenbeschränkungen

  • Einzelnes Gebäude-AssetBundle: ≤ 50 MB
  • Vorschaubild: ≤ 1 MB
  • Gesamte Download-Größe: ≤ 100 MB

Gebäude-Design-Richtlinien

1. Größenspezifikationen

Entwerfen Sie basierend auf der vom Gebäude belegten Gittergröße. Die Gebäudebasis (Fundament) sollte mindestens 1 Meter Länge und Breite haben:

Damit die Gitterbelegung des Gebäudes korrekt berechnet wird, wird empfohlen, ungerade Quadrate für die Gebäudebasisgröße zu verwenden, wie: 1×1, 3×3, 5×5. Gerade Quadrate und nicht-quadratische Formen werden nicht empfohlen, wie: 1×2, 2×2, 2×3, 4×4.

Höhe: Keine strikte Begrenzung, aber es wird empfohlen, nicht unter 0,1 Meter zu gehen

2. Visueller Stil

Um die Konsistenz des Spiels zu wahren, wird empfohlen:

  • LowPoly-Stil verwenden
  • Die Gesamtproportionen und Skalierungsverhältnisse des Gebäudes sicherstellen (Spiel-Standardskalierungsreferenz: Charakterhöhe 0,1 Meter)
  • Isometriefreundliche Designs verwenden
  • Übermäßig kleine Details vermeiden (aus der Ferne schwer zu sehen)
  • Klare Umrisse und Farbkontraste verwenden
  • Details aus verschiedenen Winkeln berücksichtigen

3. Leistungsoptimierungsempfehlungen

  • Polygonanzahl angemessen kontrollieren
  • Transparente Materialien (Alpha Test statt Alpha Blend verwenden)

Erstellungsprozess

Schritt 1: Modellierung in 3D-Software

  1. Modelle mit Blender, Maya, 3ds Max usw. erstellen
  2. Sicherstellen, dass der Modell-Ursprung in der Mitte unten ist
    • ✅ Richtig: Gebäudemittelpunkt am Weltursprung
    • ❌ Falsch: Gebäude versetzt oder Rotation nicht ausgerichtet
  3. Alle Transformationen anwenden (Scale, Rotation)
  4. Als FBX-Format exportieren

Schritt 2: Import in Unity

  1. Die FBX in das Unity-Projekt ziehen
  2. Die Transform-Position des Gebäude-Prefabs auf (0, 0, 0) setzen
  3. Sicherstellen, dass die Gebäudebasis auf der Y=0-Ebene ist
  4. Das Gebäude sollte in die positive Z-Achse zeigen

Schritt 3: Materialien und Texturen festlegen

  1. Materialien erstellen (mit URP Lit Shader)
  2. Materialparameter anpassen

Schritt 4: Erforderliche Komponenten hinzufügen

Sicherstellen, dass das Gebäude-Prefab die folgenden drei erforderlichen Komponenten enthält:

1. Mesh Filter und Mesh Renderer

  1. Das Gebäude-Root-Objekt auswählen
  2. Bestätigen, dass die Mesh Filter-Komponente hinzugefügt wurde (falls nicht, auf Add Component → Mesh Filter klicken)
  3. Bestätigen, dass die Mesh Renderer-Komponente hinzugefügt wurde
  4. Materialien im Mesh Renderer zuweisen (Standard oder URP Lit Shader verwenden)
⚠️ Wichtig: Gebäude ohne Mesh Renderer werden nicht angezeigt und lösen keine Maus-Hover-Hervorhebung aus!

2. Collider

  1. Das Gebäude-Root-Objekt auswählen
  2. Eine Box Collider-Komponente hinzufügen (empfohlen) oder Mesh Collider
  3. Die Collider-Größe anpassen, um das gesamte Gebäude abzudecken
  4. Sicherstellen, dass der Collider nicht über den Gebäudebereich hinausragt, da dies zu Kollisionen mit anderen Gebäuden führt
⚠️ Wichtig: Collider ist eine notwendige Bedingung für Maus-Hover-Hervorhebung und Klick-Erkennung! Gebäude ohne Collider können nicht von Spielern ausgewählt oder interagiert werden.

Schritt 5: Prefab erstellen

  1. Das Gebäude in das Project-Fenster ziehen, um ein Prefab zu erstellen
  2. Benennungskonvention: {BuildingName}_Prefab
  3. Testen, ob das Prefab normal instanziiert werden kann

Schritt 6: BuildingData erstellen

  1. Rechtsklick → Create → City Builder → Building Data
  2. Folgende Felder ausfüllen:
Feld Beschreibung Beispiel
Building Name Anzeigename des Gebäudes, kann Name oder Nummer verwenden Ferriswheel, SSmbl2
Building ID Eindeutiger Identifikator, empfohlen: Stilname + Nummer, oder reine Nummer SunsetCity_01, SSmbl2
Style ⭐ Bestimmt, unter welchem Stil-Tab das Gebäude im Spielbau-Menü erscheint, empfohlen: Entwicklername, Paketname oder Themenname. Gebäude derselben Serie sollten denselben Wert haben Lyon'sPack, SunsetCity
Category ⭐ Bestimmt, unter welcher Kategoriespalte es unter diesem Stil erscheint, kann bestehende Spielkategorien für Kompatibilität verwenden, oder nach Gebäudeklassifizierung ausfüllen Large, Residential
Grid Size Gitterbelegungsgröße des Gebäudes 3x3
Can Stack Für Flugzeuge oder schwebende Objekte. Wenn aktiviert, kann dieses Element in Zellen platziert werden, die bereits andere Gebäude enthalten —
Building Prefab Ihr Gebäude-Prefab hineinziehen —

Integrierte Spielkategorie-Referenz:

Kategorie-Wert Beschreibung
LargeGroß
MediumMittel
SmallKlein
⚠️ Style- und Category-Felder sind sehr wichtig: Sie werden in metadata.json verpackt (automatisch vom Upload-Tool geschrieben). Wenn das Spiel Ihren Mod lädt, bestimmt es die Position des Gebäudes im Menü basierend auf diesen beiden Feldern. Bitte stellen Sie sicher, dass Sie sie ausfüllen, sonst wird das Gebäude möglicherweise nicht in der richtigen Kategorie angezeigt.
  1. Gebäude-Symbol erstellen (empfohlen 128×128 PNG) und in das Icon-Feld ziehen

Schritt 7: Gebäude testen

  1. Das Gebäude im Spiel testen:
    • Kann es korrekt platziert werden
    • Ist die Rotation normal
    • Ist die Kollisionserkennung korrekt
    • Sind die visuellen Effekte zufriedenstellend
  2. Bis zur Zufriedenheit anpassen

Schritt 8: Vorbereitung zum Hochladen

  • Vorschaubild erstellen (empfohlen 600x600)
  • Bestätigen, dass Style und Category in BuildingData korrekt ausgefüllt sind (das Upload-Tool liest sie automatisch)

Checkliste vor dem Hochladen:

  • Modell-Dreieckszahl innerhalb der Grenzen
  • Texturauflösung angemessen
  • Enthält Mesh Filter-Komponente
  • Enthält Mesh Renderer-Komponente (Material zugewiesen)
  • Enthält Collider-Komponente
  • Prefab kann normal instanziiert werden
  • Gebäude ist am Gitter ausgerichtet
  • Keine fehlenden Materialien oder Texturen
  • Alle BuildingData-Felder ausgefüllt
  • Prüfen, ob die Datei steam_appid.txt im Projektstammverzeichnis erstellt wurde
  • Vorschaubild ist klar
  • Spielinterner Test bestanden

Schritt 9: In den Workshop hochladen

⚠️ Das Hochladen erfolgt in zwei Schritten, da AssetBundle-Packung und Steam-Upload unterschiedliche Anforderungen an den Editor-Status haben und separat bedient werden müssen.

Schritt 1: Ressourcen packen (Nicht-Play-Modus)

  1. Sicherstellen, dass Sie sich im Nicht-Play-Modus befinden (Editor läuft nicht)
  2. City Builder → Workshop Upload Tool öffnen
  3. Mod-Typ auswählen: Building
  4. BuildingData hineinziehen (das Tool liest Style und Category automatisch, kann manuell geändert werden)
  5. Das Gebäude-Prefab hineinziehen
  6. Workshop-Informationen ausfüllen (hauptsächlich Titel, Beschreibung kann im Workshop bearbeitet werden)
  7. Auf "Schritt 1: Ressourcen als AssetBundle packen" klicken
  8. Wenn die Meldung "Packung abgeschlossen! Bitte Play-Modus betreten und Schritt 2 zum Hochladen klicken" erscheint, ist es erfolgreich
💡 Tipp: Der Packungspfad wird automatisch gespeichert und geht nach dem Betreten des Play-Modus nicht verloren, keine erneute Eingabe der Informationen nötig.

Schritt 2: In Steam hochladen (Play-Modus)

  1. Den Play-Button des Unity-Editors klicken und warten, bis SteamManager initialisiert ist
  2. Im Tool-Fenster bestätigen, dass Steam initialisiert wurde (keine Warnmeldungen)
  3. Auf "Schritt 2: In Steam Workshop hochladen" klicken
  4. Warten, bis der Upload abgeschlossen ist, und die angezeigte Workshop-Item-ID notieren (für spätere Updates)
💡 Tipp: Das Upload-Tool generiert automatisch metadata.json und packt es in das AssetBundle-Inhaltsverzeichnis, das Style- und Category-Informationen enthält. Nachdem Spieler abonniert haben, zeigt das Spiel es automatisch in der richtigen Kategorie an.

Schritt 10: Veröffentlichen und Warten

  1. Informationen auf der Steam Workshop-Seite vervollständigen
  2. Weitere Screenshots hinzufügen

Hinweise zu Himmel- und Bodenmaterialien

  • Himmel- und Boden-Shader müssen die URP-Pipeline verwenden
  • Boden-Shader sollten prozedurale Normalen verwenden, um dynamische Gitterbereichsgrößenanpassungen zu bewältigen

Häufige Fehler und Lösungen

Fehler 0: "Building AssetBundles while in play mode is not allowed" beim Packen

Ursache: Der Packen-Button wurde im Play-Modus geklickt, aber AssetBundle-Packung kann nur im Nicht-Play-Modus ausgeführt werden

Lösung:

  1. Play-Modus beenden (Stopp-Button klicken)
  2. "Schritt 1: Ressourcen als AssetBundle packen" im Tool erneut klicken
  3. Nach Abschluss der Packung in den Play-Modus wechseln zum Hochladen

Fehler 1: Gebäude erscheint rosa

Ursache: Fehlendes Material oder Shader

Lösung:

  1. Sicherstellen, dass alle Texturen korrekt zugewiesen sind
  2. Prüfen, ob Materialien im AssetBundle enthalten sind

Fehler 2: Gebäude kann nicht platziert werden

Ursache: Fehlende erforderliche Komponenten (Mesh Filter/Mesh Renderer/Collider) oder falsche Gittergrößeneinstellung

Lösung:

  1. Bestätigen, dass das Gebäude-Prefab Mesh Filter, Mesh Renderer und Collider-Komponenten enthält
  2. Prüfen, ob dem Mesh Renderer Materialien zugewiesen sind
  3. Box Collider zum Gebäude-Root-Objekt hinzufügen
  4. BuildingData's Grid Size-Einstellung prüfen
  5. Gebäudeabmessungen bestätigen

Fehler 3: Gebäudeposition versetzt

Ursache: Falsche Prefab-Transform-Einstellungen

Lösung:

  1. Prefab-Position auf (0, 0, 0) setzen
  2. Sicherstellen, dass die Gebäudebasis auf der Y=0-Ebene ist
  3. Die Pivot Point-Position des Modells prüfen

Fehler 4: Gebäude nach Rotation falsch ausgerichtet

Ursache: Gebäudemittelpunkt nicht im Gitterzentrum

Lösung:

  1. Modell-Ursprung in 3D-Software anpassen
  2. Oder Position von Kind-Objekten in Unity anpassen
  3. Sicherstellen, dass das Gebäude um den Mittelpunkt rotiert

Fehler 5: Datei zu groß zum Hochladen

Ursache: Texturauflösung zu hoch oder Modell zu komplex

Lösung:

  1. Texturauflösung reduzieren (z.B. von 4K auf 2K)
  2. Modell optimieren, um Polygonanzahl zu reduzieren
  3. Texturen komprimieren (DXT5- oder BC7-Format verwenden)
  4. Unnötige Details entfernen

Erweiterte Techniken

Animation hinzufügen

Wenn Ihr Gebäude Animation enthält (z.B. rotierende Windmühle):

  1. Animation-Controller erstellen und Animation hineinziehen
  2. Animator-Komponente zum Prefab hinzufügen
  3. Animation-Controller in die Animator-Komponente ziehen

LOD-System verwenden

Detail-Culling-Skript für große Gebäude hinzufügen:

  1. BuildingDetailCulling-Skript hinzufügen
  2. Verschiedene LOD-Stufen dem Modell zuweisen
  3. Unterschiedliche Culling-Distanzen für jede Stufe festlegen

Benutzerdefinierte Materialeffekte

Mit Shader Graph spezielle Effekte erstellen:

  1. Shader Graph-Asset erstellen
  2. Benutzerdefinierte Effekte designen (z.B. leuchtende Fenster)
  3. Material mit diesem Shader erstellen
  4. Sicherstellen, dass der Shader mit der Zielplattform kompatibel ist
