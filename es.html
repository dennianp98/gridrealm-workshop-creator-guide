🏗️ Guía de Creación de Edificios para el Workshop

Versión: 1.0 | Última actualización: 2026 | Para: Unity 2022.3 LTS

¡Bienvenido, Creador! ¡Gracias por tu interés en crear contenido personalizado! ¡Tu contribución ayudará a que la comunidad prospere! Esta guía te ayudará a crear contenido que cumpla con las especificaciones y subirlo al Workshop de Steam.

Requisitos Técnicos

La subida de contenido depende de dos scripts: BuildingData y WorkshopUploadTool. BuildingData se utiliza para rellenar los datos del edificio, y WorkshopUploadTool se utiliza para la subida. También necesitas Steamworks.NET y el archivo steam_appid.txt. El archivo steam_appid.txt debe colocarse en el directorio raíz del proyecto Unity con el ID del juego: 4490290. Si necesitas actualizar Steamworks.NET, puedes descargarlo desde la página de GitHub:

https://github.com/rlabrecque/Steamworks.NET/releases

Impórtalo a Unity y adjunta el script Steam Manager a un GameObject vacío.

💡 Consejo: He empaquetado los scripts necesarios y Steamworks.NET en un archivo de paquete Unity. Puedes usar este archivo de paquete o descargarlos por separado.

Especificaciones del Modelo de Edificio

⚠️ Recordatorio de Componentes Requeridos: Los prefabs de edificio deben contener los siguientes tres componentes principales. La falta de cualquiera de ellos causará problemas de funcionamiento:
  • Mesh Filter: Muestra la malla del modelo
  • Mesh Renderer: Renderiza el modelo (requiere asignación de material)
  • Collider (Box Collider): Se usa para el resaltado al pasar el mouse, detección de clics y detección de colisión de colocación

Formatos de Recursos Soportados

  • Modelos: FBX
  • Texturas: PNG
  • Materiales: Unity URP Lit Shader

Límites de Tamaño de Archivo

  • AssetBundle de edificio individual: ≤ 50 MB
  • Imagen de vista previa: ≤ 1 MB
  • Tamaño total de descarga: ≤ 100 MB

Directrices de Diseño de Edificios

1. Especificaciones de Tamaño

Diseña según el tamaño de cuadrícula ocupado por el edificio. El tamaño de la base del edificio (cimiento) debe ser de al menos 1 metro de largo y ancho:

Para que la ocupación de cuadrícula del edificio se calcule correctamente, se recomienda usar cuadrados impares para el tamaño de la base del edificio, como: 1×1, 3×3, 5×5. No se recomiendan los cuadrados pares y las formas no cuadradas, como: 1×2, 2×2, 2×3, 4×4.

Altura: Sin límite estricto, pero se recomienda no menos de 0,1 metros

2. Estilo Visual

Para mantener la coherencia del juego, se recomienda:

  • Usar estilo LowPoly
  • Asegurar las proporciones generales y las relaciones de escala del edificio (referencia de escala predeterminada del juego: altura del personaje 0,1 metros)
  • Usar diseños amigables para vista isométrica
  • Evitar detalles excesivamente pequeños (difíciles de ver a distancia)
  • Usar contornos claros y contraste de color
  • Considerar los detalles desde diferentes ángulos

3. Recomendaciones de Optimización de Rendimiento

  • Controlar razonablemente el conteo de polígonos
  • Materiales transparentes (usar Alpha Test en lugar de Alpha Blend)

Proceso de Creación

Paso 1: Modelado en Software 3D

  1. Crear modelos usando Blender, Maya, 3ds Max, etc.
  2. Asegurar que el origen del modelo esté en el centro inferior
    • ✅ Correcto: Punto central del edificio en el origen del mundo
    • ❌ Incorrecto: Edificio desplazado o rotación no alineada
  3. Aplicar todas las transformaciones (Scale, Rotation)
  4. Exportar en formato FBX

Paso 2: Importar a Unity

  1. Arrastrar el FBX al proyecto de Unity
  2. Establecer la Transform Position del prefab de edificio en (0, 0, 0)
  3. Asegurar que la base del edificio esté en el plano Y=0
  4. El edificio debe orientarse hacia la dirección positiva del eje Z

Paso 3: Establecer Materiales y Texturas

  1. Crear materiales (usando URP Lit Shader)
  2. Ajustar parámetros del material

Paso 4: Agregar Componentes Requeridos

Asegurar que el prefab de edificio contenga los siguientes tres componentes requeridos:

1. Mesh Filter y Mesh Renderer

  1. Seleccionar el objeto raíz del edificio
  2. Confirmar que se ha agregado el componente Mesh Filter (si no, hacer clic en Add Component → Mesh Filter)
  3. Confirmar que se ha agregado el componente Mesh Renderer
  4. Asignar materiales en el Mesh Renderer (usar Standard o URP Lit Shader)
⚠️ Importante: ¡Los edificios sin Mesh Renderer no se mostrarán y no activarán el efecto de resaltado al pasar el mouse!

2. Collider

  1. Seleccionar el objeto raíz del edificio
  2. Agregar un componente Box Collider (recomendado) o Mesh Collider
  3. Ajustar el tamaño del Collider para cubrir todo el edificio
  4. Asegurar que el Collider no se extienda más allá del rango del edificio, ya que esto causaría colisiones con otros edificios
⚠️ Importante: ¡El Collider es una condición necesaria para el resaltado al pasar el mouse y la detección de clics! Los edificios sin Collider no pueden ser seleccionados o interactuados por los jugadores.

Paso 5: Crear Prefab

  1. Arrastrar el edificio a la ventana Project para crear un Prefab
  2. Convención de nomenclatura: {BuildingName}_Prefab
  3. Probar que el prefab pueda instanciarse normalmente

Paso 6: Crear BuildingData

  1. Clic derecho → Create → City Builder → Building Data
  2. Rellenar los siguientes campos:
Campo Descripción Ejemplo
Building Name Nombre de visualización del edificio, puede usar nombre o número Ferriswheel, SSmbl2
Building ID Identificador único, recomendado: nombre de estilo + número, o número puro SunsetCity_01, SSmbl2
Style ⭐ Determina en qué pestaña de estilo aparece el edificio en el menú de construcción del juego, recomendado: nombre del desarrollador, nombre del paquete o nombre del tema. Los edificios de la misma serie deben tener el mismo valor Lyon'sPack, SunsetCity
Category ⭐ Determina en qué columna de categoría aparece bajo ese estilo, puede usar categorías de juego existentes para compatibilidad, o rellenar según la clasificación de tu edificio Large, Residential
Grid Size Tamaño de ocupación de cuadrícula del edificio 3x3
Building Prefab Arrastrar tu prefab de edificio —

⚠️ Los campos Style y Category son muy importantes: Se empaquetarán en metadata.json (escrito automáticamente por la herramienta de subida). Cuando el juego carga tu mod, determina la posición del edificio en el menú basándose en estos dos campos. Asegúrate de rellenarlos, o el edificio podría no mostrarse en la categoría correcta.
  1. Crear un icono de edificio (recomendado 128×128 PNG) y arrastrarlo al campo Icon

Paso 7: Probar el Edificio

  1. Probar el edificio en el juego:
    • ¿Puede colocarse correctamente?
    • ¿La rotación es normal?
    • ¿La detección de colisión es correcta?
    • ¿Los efectos visuales son satisfactorios?
  2. Ajustar hasta estar satisfecho

Paso 8: Preparar para Subir

  • Crear una imagen de vista previa (recomendado 600x600)
  • Confirmar que Style y Category en BuildingData estén correctamente rellenados (la herramienta de subida los lee automáticamente)

Lista de verificación antes de subir:

  • El conteo de triángulos del modelo está dentro de los límites
  • La resolución de textura es razonable
  • Contiene el componente Mesh Filter
  • Contiene el componente Mesh Renderer (material asignado)
  • Contiene el componente Collider
  • El prefab puede instanciarse normalmente
  • El edificio está alineado a la cuadrícula
  • No hay materiales o texturas faltantes
  • Todos los campos de BuildingData están rellenados
  • Verificar si el archivo steam_appid.txt se ha creado en el directorio raíz del proyecto
  • La imagen de vista previa es clara
  • Prueba en el juego superada

Paso 9: Subir al Workshop

⚠️ La subida se completa en dos pasos, porque el empaquetado de AssetBundle y la subida a Steam tienen diferentes requisitos para el estado del editor y deben operarse por separado.

Paso 1: Empaquetar Recursos (Modo No-Play)

  1. Asegurar de estar actualmente en Modo No-Play (editor no ejecutándose)
  2. Abrir City Builder → Workshop Upload Tool
  3. Seleccionar tipo de mod: Building
  4. Arrastrar BuildingData (la herramienta lee automáticamente Style y Category, puede modificarse manualmente)
  5. Arrastrar el prefab de edificio
  6. Rellenar información del workshop (principalmente título, descripción puede editarse en el Workshop)
  7. Hacer clic en "Paso 1: Empaquetar recursos como AssetBundle"
  8. Cuando veas el mensaje "¡Empaquetado completo! Por favor entra en Modo Play y haz clic en el Paso 2 para subir", es exitoso
💡 Consejo: La ruta de empaquetado se guardará automáticamente y no se perderá después de entrar en Modo Play, no es necesario volver a rellenar la información.

Paso 2: Subir a Steam (Modo Play)

  1. Hacer clic en el botón Play del editor de Unity y esperar a que SteamManager se inicialice
  2. En la ventana de la herramienta, confirmar que Steam se ha inicializado (sin advertencias)
  3. Hacer clic en "Paso 2: Subir a Steam Workshop"
  4. Esperar a que la subida se complete y registrar el ID de elemento de Workshop mostrado (para actualizaciones posteriores)
💡 Consejo: La herramienta de subida genera automáticamente metadata.json y lo empaqueta en el directorio de contenido AssetBundle, que contiene la información de Style y Category. Después de que los jugadores se suscriban, el juego lo mostrará automáticamente en la categoría correcta.

Paso 10: Publicar y Mantener

  1. Completar la información en la página de Steam Workshop
  2. Agregar más capturas de pantalla

Notas sobre Materiales de Cielo y Suelo

  • Los shaders de cielo y suelo deben usar el pipeline URP
  • El shader de suelo debería usar normales procedurales para manejar ajustes dinámicos de tamaño de área de cuadrícula

Errores Comunes y Soluciones

Error 0: "Building AssetBundles while in play mode is not allowed" al empaquetar

Causa: Se hizo clic en el botón de empaquetar en Modo Play, pero el empaquetado de AssetBundle solo puede ejecutarse en Modo No-Play

Solución:

  1. Salir del Modo Play (hacer clic en el botón de detener)
  2. Volver a hacer clic en "Paso 1: Empaquetar recursos como AssetBundle" en la herramienta
  3. Entrar en Modo Play para subir después de que el empaquetado esté completo

Error 1: El edificio aparece rosa

Causa: Material o shader faltante

Solución:

  1. Asegurar que todas las texturas estén correctamente asignadas
  2. Verificar si los materiales están incluidos en el AssetBundle

Error 2: El edificio no puede colocarse

Causa: Componentes requeridos faltantes (Mesh Filter/Mesh Renderer/Collider) o configuración de tamaño de cuadrícula incorrecta

Solución:

  1. Confirmar que el prefab de edificio contenga los componentes Mesh Filter, Mesh Renderer y Collider
  2. Verificar si el Mesh Renderer tiene materiales asignados
  3. Agregar Box Collider al objeto raíz del edificio
  4. Verificar la configuración Grid Size de BuildingData
  5. Confirmar las dimensiones del edificio

Error 3: Posición del edificio desplazada

Causa: Configuraciones Transform del prefab incorrectas

Solución:

  1. Establecer la Position del prefab en (0, 0, 0)
  2. Asegurar que la base del edificio esté en el plano Y=0
  3. Verificar la posición del Pivot Point del modelo

Error 4: Edificio desalineado después de rotar

Causa: El punto central del edificio no está en el centro de la cuadrícula

Solución:

  1. Ajustar el origen del modelo en el software 3D
  2. O ajustar la posición de los objetos hijos en Unity
  3. Asegurar que el edificio rote alrededor del punto central

Error 5: Archivo demasiado grande para subir

Causa: Resolución de textura demasiado alta o modelo demasiado complejo

Solución:

  1. Reducir la resolución de textura (ej. de 4K a 2K)
  2. Optimizar el modelo para reducir el conteo de polígonos
  3. Comprimir texturas (usar formato DXT5 o BC7)
  4. Eliminar detalles innecesarios

Técnicas Avanzadas

Agregar Animación

Si tu edificio contiene animación (como un molino de viento giratorio):

  1. Crear un controlador de animación y arrastrar la animación dentro
  2. Agregar componente Animator al prefab
  3. Arrastrar el controlador de animación al componente Animator

Usar Sistema LOD

Agregar script de eliminación de detalles para edificios grandes:

  1. Agregar script BuildingDetailCulling
  2. Asignar diferentes niveles LOD al modelo
  3. Establecer diferentes distancias de eliminación para cada nivel

Efectos de Material Personalizados

Usar Shader Graph para crear efectos especiales:

  1. Crear asset Shader Graph
  2. Diseñar efectos personalizados (como ventanas luminosas)
  3. Crear material usando este Shader
  4. Asegurar que el Shader sea compatible con la plataforma objetivo
