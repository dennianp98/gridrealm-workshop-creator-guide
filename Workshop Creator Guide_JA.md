# ワークショップ建物作成ガイド

クリエイターの皆様、ようこそ！カスタムコンテンツの作成にご興味をお持ちいただきありがとうございます！あなたの貢献がコミュニティの繁栄に役立ちます！このガイドは、仕様に準拠したコンテンツを作成し、Steamワークショップにアップロードする方法を説明します。

## 技術要件
コンテンツのアップロードには、BuildingDataとWorkshopUploadToolの2つのスクリプトが必要です。BuildingDataは建物データの入力に使用し、WorkshopUploadToolはアップロードに使用します。また、Steamworks.NETも必要です。Steamworks.NETを更新する必要がある場合は、GitHubページ：https://github.com/rlabrecque/Steamworks.NET/releasesからダウンロードし、Unityにインポートして、空のゲームオブジェクトにSteam Managerスクリプトをアタッチしてください。
アップロードに必要なスクリプトとSteamworks.NETをUnityパッケージファイルにまとめました。このパッケージファイルを使用するか、個別にダウンロードできます：リンクXX

### 建物モデル仕様

> ⚠️ **必須コンポーネントのお知らせ**: 建物プレハブには以下の3つのコアコンポーネントが必須です。いずれかが欠けると機能異常が発生します：
> - **Mesh Filter**: モデルメッシュの表示
> - **Mesh Renderer**: モデルのレンダリング（マテリアルの割り当てが必要）
> - **Collider** (Box Collider): マウスホバーハイライト、クリック検出、配置衝突判定に使用

### サポートするリソース形式

- **モデル**: FBX
- **テクスチャ**: PNG
- **マテリアル**: Unity URP Lit Shader

### ファイルサイズ制限

- 単一建物AssetBundle: ≤ 50MB
- プレビュー画像: ≤ 1MB
- 総ダウンロードサイズ: ≤ 100MB

## 建物デザインガイドライン

### 1. サイズ仕様

建物が占有するグリッドサイズに基づいて設計します。建物の底部（基礎）サイズは最小でも縦横1メートル以上にしてください：
建物のグリッド占有率が正しく計算されるように、建物の底部サイズは奇数の正方形を使用することを推奨します。例：1×1、3×3、5×5。偶数の正方形や非正方形は推奨しません。例：1×2、2×2、2×3、4×4。
高さ: 厳格な制限はありませんが、最低0.1メートル以上を推奨します

### 2. ビジュアルスタイル

ゲームの一貫性を保つため、以下を推奨します：
- LowPolyスタイルを使用
- 建物の全体的なプロポーションとスケール関係を確保（ゲームデフォルトスケール参照：キャラクターの高さ0.1メートル）
- アイソメトリックビューに適したデザインを使用
- 过度に細かいディテールを避ける（遠距離では見えない）
- 明確な輪郭と色のコントラストを使用
- 異なる角度からのディテールに配慮

### 3. パフォーマンス最適化の推奨事項

- ポリゴン数を適切に管理
- 透明マテリアル（Alpha BlendではなくAlpha Testを使用）

## 作成プロセス

### ステップ1: 3Dソフトウェアでモデリング

1. Blender、Maya、3ds Maxなどを使用してモデルを作成
2. モデルの原点が底部の中心にあることを確認
Blenderを例にすると：
✅ 正しい: 建物の中心点がワールド原点にある
((画像1))
❌ 間違い: 建物がオフセットしているか回転が整列していない
((画像2))
3. すべてのトランスフォームを適用（Scale、Rotation）
4. FBX形式でエクスポート

### ステップ2: Unityにインポート

1. FBXをUnityプロジェクトにドラッグ
2. 建物プレハブのTransform Positionを(0, 0, 0)に設定
3. 建物の底部がY=0平面にあることを確認
4. 建物がZ軸正の方向を向くように設定

### ステップ3: マテリアルとテクスチャの設定

1. マテリアルを作成（URP Lit Shaderを使用）
2. マテリアルパラメータを調整

### ステップ4: 必須コンポーネントの追加

建物プレハブに以下の3つの**必須コンポーネント**が含まれていることを確認：

#### 1. Mesh FilterとMesh Renderer

1. 建物のルートオブジェクトを選択
2. **Mesh Filter**コンポーネントが追加されていることを確認（ない場合はAdd Component → Mesh Filterをクリック）
3. **Mesh Renderer**コンポーネントが追加されていることを確認
4. Mesh Rendererでマテリアルを割り当て（StandardまたはURP Lit Shaderを使用）

> ⚠️ **重要**: Mesh Rendererがない建物は表示されず、マウスホバーハイライト効果もトリガーされません！

#### 2. Collider

1. 建物のルートオブジェクトを選択
2. **Box Collider**コンポーネントを追加（推奨）または**Mesh Collider**
3. Colliderのサイズを建物全体をカバーするように調整
4. Colliderが建物の範囲を超えないように注意。超えると他の建物と衝突して配置できなくなります
((画像3))

> ⚠️ **重要**: Colliderはマウスホバーハイライトとクリック検出に必須です！Colliderがない建物はプレイヤーが選択したり操作したりできません。

### ステップ5: プレハブの作成

1. 建物をProjectウィンドウにドラッグしてPrefabを作成
2. 命名規則: `{BuildingName}_Prefab`
3. プレハブが正常にインスタンス化できるかテスト

### ステップ6: BuildingDataの作成

1. 右クリック → **Create → City Builder → Building Data**
2. 以下のフィールドを入力：

| **Building Name** | 建物の表示名、名前または番号を使用可能 | 例：Ferriswheel、SSmbl2 |
| **Building ID** | 一意の識別子、スタイル名と番号の組み合わせまたは純粋な番号を推奨 | 例：SunsetCity_01、SSmbl2 |
| **Style** ⭐ | **ゲームの建物メニューでどのスタイルタブに表示されるかを決定**、開発者名、パッケージ名、またはテーマ名の使用を推奨。同じシリーズの建物は同じ値を入力 | 例：Lyon'sPack、SunsetCity |
| **Category** ⭐ | **そのスタイル内でどのカテゴリ列に表示されるかを決定**、ゲーム内蔵カテゴリを使用して互換性を保証（下の表を参照）または独自の建物分類に従って入力 | 例：Large、Residential |
| **Grid Size** | 建物のグリッド占有サイズ | 例：3x3 |
| **Building Prefab** | 建物プレハブをドラッグ | — |

**ゲーム内蔵カテゴリ参照**:

| `Large` | 大型 |
| `Medium` | 中型 |
| `Small` | 小型 |

> ⚠️ **StyleとCategoryフィールドは非常に重要**: これらは`metadata.json`にパッケージングされます（アップロードツールが自動的に書き込み）。ゲームがモッドをロードする際、これら2つのフィールドを使用してメニュー内の建物の位置を決定します。必ず入力してください。そうしないと建物が正しく分類されて表示されない可能性があります。

3. 建物アイコンを作成（128×128 PNG推奨）、**Icon**フィールドにドラッグ

### ステップ7: 建物のテスト

1. ゲーム内で建物をテスト：
   - 正しく配置できるか
   - 回転が正常に動作するか
   - 衝突検出が正しいか
   - ビジュアル効果に満足しているか
2. 満足するまで調整

### ステップ8: アップロードの準備

- プレビュー画像を作成（600×600推奨）
- BuildingDataの**Style**と**Category**が正しく入力されていることを確認（アップロードツールが自動的に読み取ります）
アップロード前チェックリスト:
   - [ ] モデルの三角面数が制限内
   - [ ] テクスチャ解像度が適切
   - [ ] **Mesh Filter**コンポーネントを含む
   - [ ] **Mesh Renderer**コンポーネントを含む（マテリアル割り当て済み）
   - [ ] **Collider**コンポーネントを含む
   - [ ] プレハブが正常にインスタンス化できる
   - [ ] 建物がグリッドに整列している
   - [ ] 欠落しているマテリアルやテクスチャがない
   - [ ] BuildingDataのすべてのフィールドが入力済み
   - [ ] プレビュー画像が鮮明
   - [ ] ゲーム内テストに合格

## ステップ9: ワークショップにアップロード

> ⚠️ **アップロードは2つのステップで完了します**、AssetBundleのパッケージングとSteamへのアップロードはエディタの状態に対して異なる要件を持つため、別々に行う必要があります。

### ステップ1: リソースのパッケージング（非Playモード）

1. 現在**非Playモード**であることを確認（エディタが実行されていない）
2. **City Builder → Workshop Upload Tool**を開く
3. モッドタイプを選択：**Building**
4. **BuildingData**をドラッグ（ツールが自動的にStyleとCategoryを読み取り、手動で変更可能）
5. **建物プレハブ**をドラッグ
6. ワークショップ情報を入力（主にタイトルを入力、説明はワークショップで編集可能）
7. **「ステップ1: リソースをAssetBundleとしてパッケージ」**をクリック
8. **「パッケージ完了！Playモードに入ってステップ2をクリックしてアップロードしてください」**というメッセージが表示されれば成功

> 💡 パッケージングパスは自動的に保存され、Playモードに入っても失われないため、情報を再入力する必要はありません。

### ステップ2: Steamにアップロード（Playモード）

1. Unityエディタの**実行（Play）**ボタンをクリック、SteamManagerの初期化を待つ
2. ツールウィンドウでSteamが初期化されたことを確認（警告メッセージなし）
3. **「ステップ2: Steamワークショップにアップロード」**をクリック
4. アップロード完了を待ち、表示された**ワークショップアイテムID**を記録（今後の更新用）

> 💡 アップロードツールは`metadata.json`を自動的に生成し、AssetBundleコンテンツディレクトリにパッケージングします。これにはStyleとCategory情報が含まれており、プレイヤーがサブスクライブした後、ゲームは自動的に正しいカテゴリに表示します。

### ステップ10: 公開とメンテナンス

1. Steamワークショップページで情報を充実させる
2. さらなるスクリーンショットを追加

## 空と地面のマテリアルに関する注意事項
- 空と地面のシェーダーはURPパイプラインを使用する必要があります
- 地面のシェーダーは、動的なグリッドエリアサイズ調整に対応するため、プロシージャル法線を使用することを推奨します

## 一般的なエラーと解決策

### エラー0: パッケージング時のエラー "Building AssetBundles while in play mode is not allowed"

**原因**: Playモードでパッケージボタンをクリックしたが、AssetBundleのパッケージングは非Playモードでのみ実行可能

**解決策**:
1. Playモードを終了（停止ボタンをクリック）
2. ツールの「ステップ1: リソースをAssetBundleとしてパッケージ」を再度クリック
3. パッケージ完了後にPlayモードに入ってアップロード

### エラー1: 建物がピンク色で表示される

**原因**: マテリアルまたはシェーダーの欠落

**解決策**:
1. すべてのテクスチャが正しく割り当てられていることを確認
2. マテリアルがAssetBundleに含まれているか確認

### エラー2: 建物を配置できない

**原因**: 必須コンポーネントの欠落（Mesh Filter/Mesh Renderer/Collider）またはグリッドサイズ設定のエラー

**解決策**:
1. 建物プレハブが**Mesh Filter**、**Mesh Renderer**、**Collider**の3つのコンポーネントを含んでいることを確認
2. Mesh Rendererにマテリアルが割り当てられているか確認
3. 建物のルートオブジェクトにBox Colliderを追加
4. BuildingDataのGrid Size設定を確認
5. 建物の寸法を確認

### エラー3: 建物の位置がずれている

**原因**: プレハブのTransform設定が正しくない

**解決策**:
1. プレハブのPositionを(0, 0, 0)に設定
2. 建物の底部がY=0平面にあることを確認
3. モデルのPivot Pointの位置を確認

### エラー4: 回転後に建物がずれる

**原因**: 建物の中心点がグリッドの中心にない

**解決策**:
1. 3Dソフトウェアでモデルの原点を調整
2. またはUnityで子オブジェクトの位置を調整
3. 建物が中心点を中心に回転することを確認

### エラー5: ファイルが大きすぎてアップロードできない

**原因**: テクスチャ解像度が高すぎるかモデルが複雑すぎる

**解決策**:
1. テクスチャ解像度を下げる（例：4Kから2Kへ）
2. モデルを最適化してポリゴン数を減らす
3. テクスチャを圧縮（DXT5またはBC7形式を使用）
4. 不要なディテールを削除

## 高度なテクニック

### アニメーションの追加

建物にアニメーションが含まれる場合（例：回転する風車）：
1. アニメーターコントローラーを作成し、アニメーションをコントローラーにドラッグ
2. プレハブにAnimatorコンポーネントを追加
3. アニメーターコントローラーをAnimatorコンポーネントにドラッグ

### LODシステムの使用

大型建物にディテールカリングスクリプトをアタッチ：
1. BuildingDetailCullingスクリプトを追加
2. モデルに異なるLODレベルを割り当て
3. 各レベルのカリング距離を設定

### カスタムマテリアルエフェクト

Shader Graphを使用して特殊効果を作成：
1. Shader Graphアセットを作成
2. カスタムエフェクトをデザイン（例：光る窓）
3. そのシェーダーを使用するマテリアルを作成
4. シェーダーがターゲットプラットフォームと互換性があることを確認
