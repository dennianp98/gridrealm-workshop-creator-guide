🏗️ 워크숍 건축물 제작 가이드

버전: 1.0 | 최종 업데이트: 2026 | 지원: Unity 2022.3 LTS

창작자 여러분 환영합니다! 커스텀 콘텐츠 제작에 관심을 가져주셔서 감사합니다! 여러분의 기여가 커뮤니티의 번영에 도움이 될 것입니다! 이 가이드는 사양에 맞는 콘텐츠를 제작하고 Steam 창작마당에 업로드하는 방법을 안내합니다.

기술 요구사항

콘텐츠 업로드는 BuildingData와 WorkshopUploadTool 두 가지 스크립트에 의존합니다. BuildingData는 건축물 데이터를 입력하는 데 사용되며, WorkshopUploadTool은 업로드에 사용됩니다. 또한 Steamworks.NET이 필요합니다. Steamworks.NET을 업데이트해야 하는 경우 GitHub 페이지에서 다운로드할 수 있습니다:

https://github.com/rlabrecque/Steamworks.NET/releases

Unity에 가져온 후, 빈 게임오브젝트에 Steam Manager 스크립트를 연결하세요.

💡 팁: 업로드에 필요한 스크립트와 Steamworks.NET을 Unity 패키지 파일로 묶어 두었습니다. 이 패키지 파일을 사용하거나 별도로 다운로드할 수 있습니다.

건축물 모델 사양

⚠️ 필수 컴포넌트 알림: 건축물 프리팹은 다음 세 가지 핵심 컴포넌트를 반드시 포함해야 합니다. 하나라도 누락되면 기능에 문제가 발생합니다:
  • Mesh Filter: 모델 메시 표시
  • Mesh Renderer: 모델 렌더링 (머티리얼 할당 필요)
  • Collider (Box Collider): 마우스 호버 하이라이트, 클릭 감지 및 배치 충돌 감지에 사용

지원 리소스 형식

  • 모델: FBX
  • 텍스처: PNG
  • 머티리얼: Unity URP Lit Shader

파일 크기 제한

  • 단일 건축물 AssetBundle: ≤ 50MB
  • 미리보기 이미지: ≤ 1MB
  • 총 다운로드 크기: ≤ 100MB

건축물 디자인 가이드라인

1. 크기 사양

건축물이 차지하는 그리드 크기에 따라 설계합니다. 건축물 바닥(기초) 크기는 최소 가로세로 1미터 이상이어야 합니다:

건축물의 그리드 점유가 올바르게 계산되도록 건축물 바닥 크기는 홀수 정사각형을 사용하는 것을 권장합니다. 예: 1×1, 3×3, 5×5. 짝수 정사각형과 비정사각형은 권장하지 않습니다. 예: 1×2, 2×2, 2×3, 4×4.

높이: 엄격한 제한은 없으나 최소 0.1미터 이상을 권장합니다

2. 비주얼 스타일

게임의 일관성을 유지하기 위해 다음을 권장합니다:

  • LowPoly 스타일 사용
  • 건축물의 전체적인 비율과 스케일 관계 확인 (게임 기본 스케일 참조: 캐릭터 높이 0.1미터)
  • 등축 투영에 친화적인 디자인 사용
  • 지나치게 작은 디테일 피하기 (멀리서 보이지 않음)
  • 명확한 윤곽선과 색상 대비 사용
  • 다양한 각도에서의 디테일 고려

3. 성능 최적화 권장사항

  • 폴리곤 수 합리적으로 조절
  • 투명 머티리얼 (Alpha Blend 대신 Alpha Test 사용)

제작 프로세스

1단계: 3D 소프트웨어에서 모델링

  1. Blender, Maya, 3ds Max 등을 사용하여 모델 생성
  2. 모델 원점이 바닥 중앙에 있는지 확인
    • ✅ 올바름: 건축물 중심점이 월드 원점에 위치
    • ❌ 잘못됨: 건축물이 오프셋되거나 회전이 정렬되지 않음
  3. 모든 변환 적용 (Scale, Rotation)
  4. FBX 형식으로 낮내기

2단계: Unity로 가져오기

  1. FBX를 Unity 프로젝트로 드래그
  2. 건축물 프리팹의 Transform Position을 (0, 0, 0)으로 설정
  3. 건축물 바닥이 Y=0 평면에 있는지 확인
  4. 건축물이 Z축 양의 방향을 향하도록 설정

3단계: 머티리얼 및 텍스처 설정

  1. 머티리얼 생성 (URP Lit Shader 사용)
  2. 머티리얼 파라미터 조정

4단계: 필수 컴포넌트 추가

건축물 프리팹이 다음 세 가지 필수 컴포넌트를 포함하는지 확인:

1. Mesh Filter와 Mesh Renderer

  1. 건축물 루트 오브젝트 선택
  2. Mesh Filter 컴포넌트가 추가되었는지 확인 (없으면 Add Component → Mesh Filter 클릭)
  3. Mesh Renderer 컴포넌트가 추가되었는지 확인
  4. Mesh Renderer에서 머티리얼 할당 (Standard 또는 URP Lit Shader 사용)
⚠️ 중요: Mesh Renderer가 없는 건축물은 표시되지 않으며 마우스 호버 하이라이트 효과도 작동하지 않습니다!

2. Collider

  1. 건축물 루트 오브젝트 선택
  2. Box Collider 컴포넌트 추가 (권장) 또는 Mesh Collider
  3. Collider 크기를 건축물 전체를 덮도록 조정
  4. Collider가 건축물 범위를 벗어나지 않도록 주의. 벗어나면 다른 건축물과 충돌하여 배치할 수 없습니다
⚠️ 중요: Collider는 마우스 호버 하이라이트와 클릭 감지의 필수 조건입니다! Collider가 없는 건축물은 플레이어가 선택하거나 상호작용할 수 없습니다.

5단계: 프리팹 생성

  1. 건축물을 Project 창으로 드래그하여 Prefab 생성
  2. 명명 규칙: {BuildingName}_Prefab
  3. 프리팹이 정상적으로 인스턴스화되는지 테스트

6단계: BuildingData 생성

  1. 우클릭 → Create → City Builder → Building Data
  2. 다음 필드를 입력:
필드 설명 예시
Building Name 건축물 표시 이름, 이름 또는 번호 사용 가능 Ferriswheel, SSmbl2
Building ID 고유 식별자, 스타일 이름+번호 사용 권장, 또는 순수 번호 SunsetCity_01, SSmbl2
Style ⭐ 게임의 건축물 메뉴에서 어떤 스타일 탭에 표시될지 결정, 개발자 이름, 패키지 이름 또는 테마 이름 사용 권장. 같은 시리즈의 건축물은 같은 값을 설정 Lyon'sPack, SunsetCity
Category ⭐ 해당 스타일 아래 어떤 카테고리 열에 표시될지 결정, 호환성을 위해 기존 게임 카테고리 사용 가능, 또는 건축물 분류에 따라 입력 Large, Residential
Grid Size 건축물 그리드 점유 크기 3x3
Building Prefab 건축물 프리팹을 드래그 —

게임 내장 카테고리 참조:

카테고리 값 설명
Large대형
Medium중형
Small소형
⚠️ Style과 Category 필드는 매우 중요: 이들은 metadata.json에 패키지화됩니다 (업로드 도구가 자동으로 기록). 게임이 모드를 로드할 때, 이 두 필드를 기반으로 건축물의 메뉴 내 위치를 결정합니다. 반드시 입력하세요. 그렇지 않으면 건축물이 올바른 카테고리에 표시되지 않을 수 있습니다.
  1. 건축물 아이콘 생성 (권장 128×128 PNG) 후 Icon 필드에 드래그

7단계: 건축물 테스트

  1. 게임 내에서 건축물 테스트:
    • 올바르게 배치할 수 있는지
    • 회전이 정상적인지
    • 충돌 감지가 정확한지
    • 시각 효과가 만족스러운지
  2. 만족할 때까지 조정

8단계: 업로드 준비

  • 미리보기 이미지 생성 (권장 600x600)
  • BuildingData의 Style과 Category가 올바르게 입력되었는지 확인 (업로드 도구가 자동으로 읽기)

업로드 전 체크리스트:

  • 모델 삼각형 수가 제한 내
  • 텍스처 해상도가 적절
  • Mesh Filter 컴포넌트 포함
  • Mesh Renderer 컴포넌트 포함 (머티리얼 할당됨)
  • Collider 컴포넌트 포함
  • 프리팹이 정상적으로 인스턴스화 가능
  • 건축물이 그리드에 정렬됨
  • 누락된 머티리얼이나 텍스처 없음
  • BuildingData의 모든 필드 입력됨
  • 미리보기 이미지가 선명함
  • 게임 내 테스트 통과

9단계: 창작마당에 업로드

⚠️ 업로드는 2단계로 완료됩니다. AssetBundle 패키지화와 Steam 업로드는 에디터 상태에 대한 요구사항이 다르므로 별도로 조작해야 합니다.

1단계: 리소스 패키지화 (비Play 모드)

  1. 현재 비Play 모드인지 확인 (에디터가 실행 중이 아님)
  2. City Builder → Workshop Upload Tool 열기
  3. 모드 유형 선택: Building
  4. BuildingData를 드래그 (도구가 자동으로 Style과 Category를 읽고, 수동으로 변경 가능)
  5. 건축물 프리팹을 드래그
  6. 창작마당 정보 입력 (주로 제목, 설명은 창작마당에서 편집 가능)
  7. "1단계: 리소스를 AssetBundle으로 패키지화" 클릭
  8. "패키지화 완료! Play 모드에 진입하여 2단계를 클릭하여 업로드하세요" 프롬프트가 표시되면 성공
💡 팁: 패키지화 경로는 자동으로 저장되며, Play 모드에 진입해도 손실되지 않습니다. 정보를 다시 입력할 필요가 없습니다.

2단계: Steam에 업로드 (Play 모드)

  1. Unity 에디터의 Play 버튼을 클릭하고 SteamManager의 초기화를 기다림
  2. 도구 창에서 Steam이 초기화되었는지 확인 (경고 프롬프트 없음)
  3. "2단계: Steam 창작마당에 업로드" 클릭
  4. 업로드가 완료될 때까지 기다리고 표시된 창작마당 아이템 ID를 기록 (후속 업데이트용)
💡 팁: 업로드 도구는 자동으로 metadata.json을 생성하고 AssetBundle 콘텐츠 디렉토리에 패키지화합니다. 여기에는 Style과 Category 정보가 포함됩니다. 플레이어가 구독한 후, 게임은 자동으로 올바른 카테고리에 표시합니다.

10단계: 공개 및 유지보수

  1. Steam 창작마당 페이지에서 정보 완성
  2. 더 많은 스크린샷 추가

하늘 및 지면 머티리얼 참고사항

  • 하늘 및 지면 셰이더는 URP 파이프라인을 사용해야 함
  • 지면 셰이더는 그리드 영역의 동적 크기 조정에 대응하기 위해 프로시저럴 노멀을 사용하는 것을 권장

일반적인 오류 및 해결책

오류 0: 패키지화 시 "Building AssetBundles while in play mode is not allowed"

원인: Play 모드에서 패키지화 버튼을 클릭했지만, AssetBundle 패키지화는 비Play 모드에서만 실행 가능

해결책:

  1. Play 모드 종료 (중지 버튼 클릭)
  2. 도구의 "1단계: 리소스를 AssetBundle으로 패키지화"를 다시 클릭
  3. 패키지화 완료 후 Play 모드에 진입하여 업로드

오류 1: 건축물이 분홍색으로 표시됨

원인: 머티리얼 또는 셰이더 누락

해결책:

  1. 모든 텍스처가 올바르게 할당되었는지 확인
  2. 머티리얼이 AssetBundle에 포함되어 있는지 확인

오류 2: 건축물을 배치할 수 없음

원인: 필수 컴포넌트(Mesh Filter/Mesh Renderer/Collider) 누락 또는 그리드 크기 설정 오류

해결책:

  1. 건축물 프리팹에 Mesh Filter, Mesh Renderer, Collider 컴포넌트가 포함되어 있는지 확인
  2. Mesh Renderer에 머티리얼이 할당되었는지 확인
  3. 건축물 루트 오브젝트에 Box Collider 추가
  4. BuildingData의 Grid Size 설정 확인
  5. 건축물 크기 확인

오류 3: 건축물 위치가 어긋남

원인: 프리팹 Transform 설정 오류

해결책:

  1. 프리팹 Position을 (0, 0, 0)으로 설정
  2. 건축물 바닥이 Y=0 평면에 있는지 확인
  3. 모델의 Pivot Point 위치 확인

오류 4: 회전 후 건축물이 어긋남

원인: 건축물 중심점이 그리드 중심에 없음

해결책:

  1. 3D 소프트웨어에서 모델 원점 조정
  2. 또는 Unity에서 자식 오브젝트의 위치 조정
  3. 건축물이 중심점을 중심으로 회전하는지 확인

오류 5: 파일이 너무 커서 업로드할 수 없음

원인: 텍스처 해상도가 너무 높거나 모델이 너무 복잡함

해결책:

  1. 텍스처 해상도 낮추기 (예: 4K에서 2K로)
  2. 모델을 최적화하여 폴리곤 수 줄이기
  3. 텍스처 압축 (DXT5 또는 BC7 형식 사용)
  4. 불필요한 디테일 제거

고급 기법

애니메이션 추가

건축물에 애니메이션이 포함된 경우 (예: 회전하는 풍차):

  1. 애니메이션 컨트롤러를 생성하고 애니메이션을 드래그
  2. 프리팹에 Animator 컴포넌트 추가
  3. 애니메이션 컨트롤러를 Animator 컴포넌트에 드래그

LOD 시스템 사용

대형 건축물에 디테일 제거 스크립트 추가:

  1. BuildingDetailCulling 스크립트 추가
  2. 모델에 다른 LOD 레벨 할당
  3. 각 레벨의 제거 거리 설정

커스텀 머티리얼 효과

Shader Graph를 사용하여 특수 효과 생성:

  1. Shader Graph 에셋 생성
  2. 커스텀 효과 설계 (예: 발광하는 창문)
  3. 이 셰이더를 사용하여 머티리얼 생성
  4. 셰이더가 타겟 플랫폼과 호환되는지 확인
