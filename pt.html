🏗️ Guia de Criação de Edifícios para a Oficina

Versão: 1.0 | Última atualização: 2026 | Para: Unity 2022.3 LTS

Bem-vindo, Criador! Obrigado pelo seu interesse em criar conteúdo personalizado! Sua contribuição ajudará a comunidade a prosperar! Este guia irá ajudá-lo a criar conteúdo que atenda às especificações e carregá-lo na Oficina Steam.

Requisitos Técnicos

O carregamento de conteúdo depende de dois scripts: BuildingData e WorkshopUploadTool. BuildingData é usado para preencher os dados do edifício, e WorkshopUploadTool é usado para o carregamento. Você também precisa do Steamworks.NET. Se precisar atualizar o Steamworks.NET, pode baixá-lo da página do GitHub:

https://github.com/rlabrecque/Steamworks.NET/releases

Importe-o para o Unity e anexe o script Steam Manager a um GameObject vazio.

💡 Dica: Empacotei os scripts necessários e o Steamworks.NET em um arquivo de pacote Unity. Você pode usar este arquivo de pacote ou baixá-los separadamente.

Especificações do Modelo de Edifício

⚠️ Lembrete de Componentes Obrigatórios: Os prefabs de edifício devem conter os três componentes principais a seguir. A falta de qualquer um deles causará mau funcionamento:
  • Mesh Filter: Exibe a malha do modelo
  • Mesh Renderer: Renderiza o modelo (requer atribuição de material)
  • Collider (Box Collider): Usado para destaque ao passar o mouse, detecção de clique e detecção de colisão de posicionamento

Formatos de Recursos Suportados

  • Modelos: FBX
  • Texturas: PNG
  • Materiais: Unity URP Lit Shader

Limites de Tamanho de Arquivo

  • AssetBundle de edifício único: ≤ 50 MB
  • Imagem de visualização: ≤ 1 MB
  • Tamanho total de download: ≤ 100 MB

Diretrizes de Design de Edifícios

1. Especificações de Tamanho

Projete com base no tamanho da grade ocupado pelo edifício. O tamanho da base do edifício (fundação) deve ser de no mínimo 1 metro de comprimento e largura:

Para que a ocupação da grade do edifício seja calculada corretamente, recomenda-se usar quadrados ímpares para o tamanho da base do edifício, como: 1×1, 3×3, 5×5. Quadrados pares e formas não quadradas não são recomendados, como: 1×2, 2×2, 2×3, 4×4.

Altura: Sem limite estrito, mas recomenda-se não inferior a 0,1 metros

2. Estilo Visual

Para manter a consistência do jogo, recomenda-se:

  • Usar estilo LowPoly
  • Garantir as proporções gerais e relações de escala do edifício (referência de escala padrão do jogo: altura do personagem 0,1 metros)
  • Usar designs amigáveis para vista isométrica
  • Evitar detalhes excessivamente pequenos (difíceis de ver à distância)
  • Usar contornos claros e contraste de cores
  • Considerar os detalhes de diferentes ângulos

3. Recomendações de Otimização de Desempenho

  • Controlar razoavelmente a contagem de polígonos
  • Materiais transparentes (usar Alpha Test em vez de Alpha Blend)

Processo de Criação

Etapa 1: Modelagem em Software 3D

  1. Criar modelos usando Blender, Maya, 3ds Max, etc.
  2. Garantir que a origem do modelo esteja no centro inferior
    • ✅ Correto: Ponto central do edifício na origem do mundo
    • ❌ Incorreto: Edifício deslocado ou rotação não alinhada
  3. Aplicar todas as transformações (Scale, Rotation)
  4. Exportar em formato FBX

Etapa 2: Importar para o Unity

  1. Arrastar o FBX para o projeto Unity
  2. Definir a Transform Position do prefab de edifício para (0, 0, 0)
  3. Garantir que a base do edifício esteja no plano Y=0
  4. O edifício deve orientar-se para a direção positiva do eixo Z

Etapa 3: Definir Materiais e Texturas

  1. Criar materiais (usando URP Lit Shader)
  2. Ajustar parâmetros do material

Etapa 4: Adicionar Componentes Obrigatórios

Garantir que o prefab de edifício contenha os seguintes três componentes obrigatórios:

1. Mesh Filter e Mesh Renderer

  1. Selecionar o objeto raiz do edifício
  2. Confirmar que o componente Mesh Filter foi adicionado (se não, clicar em Add Component → Mesh Filter)
  3. Confirmar que o componente Mesh Renderer foi adicionado
  4. Atribuir materiais no Mesh Renderer (usar Standard ou URP Lit Shader)
⚠️ Importante: Edifícios sem Mesh Renderer não serão exibidos e não acionarão o efeito de destaque ao passar o mouse!

2. Collider

  1. Selecionar o objeto raiz do edifício
  2. Adicionar um componente Box Collider (recomendado) ou Mesh Collider
  3. Ajustar o tamanho do Collider para cobrir todo o edifício
  4. Garantir que o Collider não se estenda além do alcance do edifício, pois isso causaria colisões com outros edifícios
⚠️ Importante: O Collider é uma condição necessária para o destaque ao passar o mouse e a detecção de cliques! Edifícios sem Collider não podem ser selecionados ou interagidos pelos jogadores.

Etapa 5: Criar Prefab

  1. Arrastar o edifício para a janela Project para criar um Prefab
  2. Convenção de nomenclatura: {BuildingName}_Prefab
  3. Testar se o prefab pode ser instanciado normalmente

Etapa 6: Criar BuildingData

  1. Clique direito → Create → City Builder → Building Data
  2. Preencher os seguintes campos:
Campo Descrição Exemplo
Building Name Nome de exibição do edifício, pode usar nome ou número Ferriswheel, SSmbl2
Building ID Identificador único, recomendado: nome de estilo + número, ou número puro SunsetCity_01, SSmbl2
Style ⭐ Determina em qual aba de estilo o edifício aparece no menu de construção do jogo, recomendado: nome do desenvolvedor, nome do pacote ou nome do tema. Edifícios da mesma série devem ter o mesmo valor Lyon'sPack, SunsetCity
Category ⭐ Determina em qual coluna de categoria aparece sob esse estilo, pode usar categorias de jogo existentes para compatibilidade, ou preencher de acordo com a classificação do seu edifício Large, Residential
Grid Size Tamanho de ocupação de grade do edifício 3x3
Building Prefab Arrastar seu prefab de edifício —

Referência de categorias integradas do jogo:

Valor de categoria Descrição
LargeGrande
MediumMédio
SmallPequeno
⚠️ Os campos Style e Category são muito importantes: Eles serão empacotados em metadata.json (escrito automaticamente pela ferramenta de carregamento). Quando o jogo carrega seu mod, determina a posição do edifício no menu com base nesses dois campos. Certifique-se de preenchê-los, ou o edifício pode não ser exibido na categoria correta.
  1. Criar um ícone de edifício (recomendado 128×128 PNG) e arrastá-lo para o campo Icon

Etapa 7: Testar o Edifício

  1. Testar o edifício no jogo:
    • Pode ser colocado corretamente
    • A rotação é normal
    • A detecção de colisão está correta
    • Os efeitos visuais são satisfatórios
  2. Ajustar até ficar satisfeito

Etapa 8: Preparar para Carregar

  • Criar uma imagem de visualização (recomendado 600x600)
  • Confirmar que Style e Category no BuildingData estão corretamente preenchidos (a ferramenta de carregamento os lê automaticamente)

Lista de verificação antes de carregar:

  • A contagem de triângulos do modelo está dentro dos limites
  • A resolução de textura é razoável
  • Contém o componente Mesh Filter
  • Contém o componente Mesh Renderer (material atribuído)
  • Contém o componente Collider
  • O prefab pode ser instanciado normalmente
  • O edifício está alinhado à grade
  • Não há materiais ou texturas faltantes
  • Todos os campos do BuildingData estão preenchidos
  • A imagem de visualização está clara
  • Teste no jogo aprovado

Etapa 9: Carregar para a Oficina

⚠️ O carregamento é concluído em duas etapas, porque o empacotamento de AssetBundle e o carregamento para a Steam têm requisitos diferentes para o estado do editor e devem ser operados separadamente.

Etapa 1: Empacotar Recursos (Modo Não-Play)

  1. Garantir que você está atualmente no Modo Não-Play (editor não em execução)
  2. Abrir City Builder → Workshop Upload Tool
  3. Selecionar tipo de mod: Building
  4. Arrastar BuildingData (a ferramenta lê automaticamente Style e Category, pode ser modificado manualmente)
  5. Arrastar o prefab de edifício
  6. Preencher informações da oficina (principalmente título, descrição pode ser editada na Oficina)
  7. Clicar em "Etapa 1: Empacotar recursos como AssetBundle"
  8. Quando você vir a mensagem "Empacotamento completo! Por favor entre no Modo Play e clique na Etapa 2 para carregar", é bem-sucedido
💡 Dica: O caminho de empacotamento será salvo automaticamente e não será perdido após entrar no Modo Play, não é necessário preencher as informações novamente.

Etapa 2: Carregar para a Steam (Modo Play)

  1. Clicar no botão Play do editor Unity e aguardar a inicialização do SteamManager
  2. Na janela da ferramenta, confirmar que a Steam foi inicializada (sem avisos)
  3. Clicar em "Etapa 2: Carregar para a Steam Workshop"
  4. Aguardar a conclusão do carregamento e registrar o ID de item da Oficina exibido (para atualizações posteriores)
💡 Dica: A ferramenta de carregamento gera automaticamente metadata.json e o empacota no diretório de conteúdo AssetBundle, que contém as informações de Style e Category. Depois que os jogadores se inscreverem, o jogo o exibirá automaticamente na categoria correta.

Etapa 10: Publicar e Manter

  1. Completar as informações na página da Steam Workshop
  2. Adicionar mais capturas de tela

Notas sobre Materiais de Céu e Solo

  • Os shaders de céu e solo devem usar o pipeline URP
  • O shader de solo deve usar normais procedurais para lidar com ajustes dinâmicos de tamanho de área de grade

Erros Comuns e Soluções

Erro 0: "Building AssetBundles while in play mode is not allowed" ao empacotar

Causa: O botão de empacotar foi clicado no Modo Play, mas o empacotamento de AssetBundle só pode ser executado no Modo Não-Play

Solução:

  1. Sair do Modo Play (clicar no botão de parar)
  2. Clicar novamente em "Etapa 1: Empacotar recursos como AssetBundle" na ferramenta
  3. Entrar no Modo Play para carregar após o empacotamento estar completo

Erro 1: O edifício aparece rosa

Causa: Material ou shader faltante

Solução:

  1. Garantir que todas as texturas estejam corretamente atribuídas
  2. Verificar se os materiais estão incluídos no AssetBundle

Erro 2: O edifício não pode ser colocado

Causa: Componentes obrigatórios faltantes (Mesh Filter/Mesh Renderer/Collider) ou configuração de tamanho de grade incorreta

Solução:

  1. Confirmar que o prefab de edifício contém os componentes Mesh Filter, Mesh Renderer e Collider
  2. Verificar se o Mesh Renderer tem materiais atribuídos
  3. Adicionar Box Collider ao objeto raiz do edifício
  4. Verificar a configuração Grid Size do BuildingData
  5. Confirmar as dimensões do edifício

Erro 3: Posição do edifício deslocada

Causa: Configurações Transform do prefab incorretas

Solução:

  1. Definir a Position do prefab para (0, 0, 0)
  2. Garantir que a base do edifício esteja no plano Y=0
  3. Verificar a posição do Pivot Point do modelo

Erro 4: Edifício desalinhado após girar

Causa: O ponto central do edifício não está no centro da grade

Solução:

  1. Ajustar a origem do modelo no software 3D
  2. Ou ajustar a posição dos objetos filhos no Unity
  3. Garantir que o edifício gire ao redor do ponto central

Erro 5: Arquivo muito grande para carregar

Causa: Resolução de textura muito alta ou modelo muito complexo

Solução:

  1. Reduzir a resolução de textura (ex. de 4K para 2K)
  2. Otimizar o modelo para reduzir a contagem de polígonos
  3. Comprimir texturas (usar formato DXT5 ou BC7)
  4. Remover detalhes desnecessários

Técnicas Avançadas

Adicionar Animação

Se seu edifício contém animação (como um moinho de vento giratório):

  1. Criar um controlador de animação e arrastar a animação para dentro
  2. Adicionar componente Animator ao prefab
  3. Arrastar o controlador de animação para o componente Animator

Usar Sistema LOD

Adicionar script de eliminação de detalhes para edifícios grandes:

  1. Adicionar script BuildingDetailCulling
  2. Atribuir diferentes níveis LOD ao modelo
  3. Definir diferentes distâncias de eliminação para cada nível

Efeitos de Material Personalizados

Usar Shader Graph para criar efeitos especiais:

  1. Criar asset Shader Graph
  2. Projetar efeitos personalizados (como janelas luminosas)
  3. Criar material usando este Shader
  4. Garantir que o Shader seja compatível com a plataforma alvo
