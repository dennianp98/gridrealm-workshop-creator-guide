# Руководство по созданию зданий для Мастерской

Добро пожаловать, Создатель! Спасибо за интерес к созданию пользовательского контента! Ваш вклад поможет процветанию сообщества! Это руководство поможет вам создать контент, соответствующий спецификациям, и загрузить его в Мастерскую Steam.

## Технические требования
Загрузка контента зависит от двух скриптов: BuildingData и WorkshopUploadTool. BuildingData используется для заполнения данных здания, а WorkshopUploadTool — для загрузки. Вам также понадобится Steamworks.NET. Если вам нужно обновить Steamworks.NET, вы можете скачать его со страницы GitHub: https://github.com/rlabrecque/Steamworks.NET/releases, импортировать в Unity и прикрепить скрипт Steam Manager к пустому GameObject.
Я упаковал необходимые скрипты и Steamworks.NET в файл пакета Unity. Вы можете использовать этот файл пакета или скачать их отдельно: Ссылка XX

### Спецификации модели здания

> ⚠️ **Напоминание о обязательных компонентах**: Префабы зданий должны содержать следующие три основных компонента. Отсутствие любого из них приведёт к некорректной работе:
> - **Mesh Filter**: Отображает меш модели
> - **Mesh Renderer**: Рендерит модель (требуется назначение материала)
> - **Collider** (Box Collider): Используется для подсветки при наведении мыши, обнаружения кликов и определения коллизий при размещении

### Поддерживаемые форматы ресурсов

- **Модели**: FBX
- **Текстуры**: PNG
- **Материалы**: Unity URP Lit Shader

### Ограничения размера файлов

- Один AssetBundle здания: ≤ 50 МБ
- Изображение предпросмотра: ≤ 1 МБ
- Общий размер загрузки: ≤ 100 МБ

## Руководство по дизайну зданий

### 1. Спецификации размера

Проектируйте в соответствии с размером сетки, занимаемым зданием. Размер основания здания (фундамента) должен быть минимум 1 метр в длину и ширину:
Для корректного расчёта занимаемой зданием сетки рекомендуется использовать нечётные квадраты для размера основания здания, например: 1×1, 3×3, 5×5. Чётные квадраты и неквадратные формы не рекомендуются, например: 1×2, 2×2, 2×3, 4×4.
Высота: Строгих ограничений нет, но рекомендуется не ниже 0,1 метра

### 2. Визуальный стиль

Для сохранения согласованности игры рекомендуется:
- Использовать стиль LowPoly
- Обеспечить общие пропорции и соотношения масштаба здания (справочный масштаб игры по умолчанию: высота персонажа 0,1 метра)
- Использовать дизайн, дружественный к изометрическому виду
- Избегать слишком мелких деталей (невидны издалека)
- Использовать чёткие контуры и цветовой контраст
- Учитывать детали с разных углов

### 3. Рекомендации по оптимизации производительности

- Разумно контролировать количество полигонов
- Прозрачные материалы (использовать Alpha Test вместо Alpha Blend)

## Процесс создания

### Шаг 1: Моделирование в 3D-программе

1. Создайте модели с помощью Blender, Maya, 3ds Max и т.д.
2. Убедитесь, что начало координат модели находится в нижнем центре
На примере Blender:
✅ Правильно: Центральная точка здания находится в начале мировых координат
((Изображение 1))
❌ Неправильно: Здание смещено или поворот не выровнен
((Изображение 2))
3. Примените все трансформации (Scale, Rotation)
4. Экспортируйте в формате FBX

### Шаг 2: Импорт в Unity

1. Перетащите FBX в проект Unity
2. Установите Transform Position префаба здания на (0, 0, 0)
3. Убедитесь, что основание здания находится на плоскости Y=0
4. Здание должно быть направлено в положительную сторону оси Z

### Шаг 3: Настройка материалов и текстур

1. Создайте материалы (используя URP Lit Shader)
2. Настройте параметры материалов

### Шаг 4: Добавление обязательных компонентов

Убедитесь, что префаб здания содержит следующие три **обязательных компонента**:

#### 1. Mesh Filter и Mesh Renderer

1. Выберите корневой объект здания
2. Подтвердите, что компонент **Mesh Filter** добавлен (если нет, нажмите Add Component → Mesh Filter)
3. Подтвердите, что компонент **Mesh Renderer** добавлен
4. Назначьте материалы в Mesh Renderer (используйте Standard или URP Lit Shader)

> ⚠️ **Важно**: Здания без Mesh Renderer не будут отображаться и не будут вызывать эффект подсветки при наведении мыши!

#### 2. Collider

1. Выберите корневой объект здания
2. Добавьте компонент **Box Collider** (рекомендуется) или **Mesh Collider**
3. Настройте размер Collider, чтобы он покрывал всё здание
4. Убедитесь, что Collider не выходит за пределы здания, так как это приведёт к коллизии с другими зданиями и невозможности размещения
((Изображение 3))

> ⚠️ **Важно**: Collider — необходимое условие для подсветки при наведении мыши и обнаружения кликов! Здания без Collider не могут быть выбраны или взаимодействовать с игроком.

### Шаг 5: Создание префаба

1. Перетащите здание в окно Project для создания Prefab
2. Соглашение об именовании: `{BuildingName}_Prefab`
3. Протестируйте, что префаб может нормально инстанцироваться

### Шаг 6: Создание BuildingData

1. Правый клик → **Create → City Builder → Building Data**
2. Заполните следующие поля:

| **Building Name** | Отображаемое название здания, можно использовать имя или номер | Напр.: Ferriswheel, SSmbl2 |
| **Building ID** | Уникальный идентификатор, рекомендуется использовать название стиля плюс номер или просто номер | Напр.: SunsetCity_01, SSmbl2 |
| **Style** ⭐ | **Определяет, в какой вкладке стиля здание появится в меню строительства игры**, рекомендуется использовать ваше имя разработчика, название пакета или темы. Здания одной серии должны иметь одинаковое значение | Напр.: Lyon'sPack, SunsetCity |
| **Category** ⭐ | **Определяет, в какой колонке категории оно появится под этим стилем**, можно использовать встроенные категории игры для совместимости (см. таблицу ниже) или заполнить по своей классификации зданий | Напр.: Large, Residential |
| **Grid Size** | Размер занимаемой зданием сетки | Напр.: 3x3 |
| **Building Prefab** | Перетащите ваш префаб здания | — |

**Справочник встроенных категорий игры**:

| `Large` | Большой |
| `Medium` | Средний |
| `Small` | Малый |

> ⚠️ **Поля Style и Category очень важны**: Они будут упакованы в `metadata.json` (автоматически записывается инструментом загрузки). Когда игра загружает ваш мод, она использует эти два поля для определения позиции здания в меню. Обязательно заполните их, иначе здание может не корректно категоризироваться и отображаться.

3. Создайте иконку здания (рекомендуется PNG 128×128), перетащите в поле **Icon**

### Шаг 7: Тестирование здания

1. Протестируйте здание в игре:
   - Можно ли его корректно разместить
   - Работает ли вращение нормально
   - Правильно ли работает обнаружение коллизий
   - Удовлетворены ли вы визуальным эффектом
2. Настраивайте до удовлетворения

### Шаг 8: Подготовка к загрузке

- Создайте изображение предпросмотра (рекомендуется 600×600)
- Подтвердите, что **Style** и **Category** в BuildingData корректно заполнены (инструмент загрузки считает их автоматически)
Контрольный список перед загрузкой:
   - [ ] Количество треугольников модели в пределах лимита
   - [ ] Разрешение текстур разумное
   - [ ] Содержит компонент **Mesh Filter**
   - [ ] Содержит компонент **Mesh Renderer** (материал назначен)
   - [ ] Содержит компонент **Collider**
   - [ ] Префаб может нормально инстанцироваться
   - [ ] Здание выровнено по сетке
   - [ ] Нет отсутствующих материалов или текстур
   - [ ] Все поля BuildingData заполнены
   - [ ] Изображение предпросмотра чёткое
   - [ ] Прошло тестирование в игре

## Шаг 9: Загрузка в Мастерскую

> ⚠️ **Загрузка выполняется в два этапа**, так как упаковка AssetBundle и загрузка в Steam имеют разные требования к состоянию редактора и должны выполняться отдельно.

### Первый этап: Упаковка ресурсов (режим Non-Play)

1. Убедитесь, что вы находитесь в **режиме Non-Play** (редактор не запущен)
2. Откройте **City Builder → Workshop Upload Tool**
3. Выберите тип мода: **Building**
4. Перетащите **BuildingData** (инструмент автоматически считает Style и Category, можно изменить вручную)
5. Перетащите **префаб здания**
6. Заполните информацию мастерской (в основном заголовок, описание можно редактировать в Мастерской)
7. Нажмите **"Первый этап: Упаковать ресурсы как AssetBundle"**
8. Когда увидите сообщение **"Упаковка завершена! Пожалуйста, войдите в режим Play и нажмите второй этап для загрузки"**, это успешно

> 💡 Путь упаковки будет автоматически сохранён и не потеряется после входа в режим Play, не нужно повторно вводить информацию.

### Второй этап: Загрузка в Steam (режим Play)

1. Нажмите кнопку **Play** редактора Unity, дождитесь инициализации SteamManager
2. Подтвердите в окне инструмента, что Steam инициализирован (нет предупреждающих сообщений)
3. Нажмите **"Второй этап: Загрузить в Мастерскую Steam"**
4. Дождитесь завершения загрузки, запишите отображаемый **ID элемента Мастерской** (для будущих обновлений)

> 💡 Инструмент загрузки автоматически сгенерирует `metadata.json` и упакует его в директорию содержимого AssetBundle, которая содержит информацию Style и Category. После подписки игроков игра автоматически отобразит в правильной категории.

### Шаг 10: Публикация и поддержка

1. Дополните информацию на странице Мастерской Steam
2. Добавьте больше скриншотов

## Примечания по материалам неба и земли
- Шейдеры неба и земли должны использовать пайплайн URP
- Шейдеры земли предпочтительно должны использовать процедурные нормали для обработки динамической настройки размера области сетки

## Частые ошибки и решения

### Ошибка 0: Ошибка упаковки "Building AssetBundles while in play mode is not allowed"

**Причина**: Кнопка упаковки была нажата в режиме Play, но упаковка AssetBundle может выполняться только в режиме Non-Play

**Решение**:
1. Выйдите из режима Play (нажмите кнопку остановки)
2. Снова нажмите "Первый этап: Упаковать ресурсы как AssetBundle" в инструменте
3. После завершения упаковки войдите в режим Play для загрузки

### Ошибка 1: Здание отображается розовым

**Причина**: Отсутствует материал или шейдер

**Решение**:
1. Убедитесь, что все текстуры корректно назначены
2. Проверьте, включены ли материалы в AssetBundle

### Ошибка 2: Здание нельзя разместить

**Причина**: Отсутствуют обязательные компоненты (Mesh Filter/Mesh Renderer/Collider) или неправильные настройки размера сетки

**Решение**:
1. Подтвердите, что префаб здания содержит компоненты **Mesh Filter**, **Mesh Renderer** и **Collider**
2. Проверьте, назначены ли материалы в Mesh Renderer
3. Добавьте Box Collider к корневому объекту здания
4. Проверьте настройку Grid Size в BuildingData
5. Подтвердите размеры здания

### Ошибка 3: Смещение позиции здания

**Причина**: Неправильные настройки Transform префаба

**Решение**:
1. Установите Position префаба на (0, 0, 0)
2. Убедитесь, что основание здания находится на плоскости Y=0
3. Проверьте положение Pivot Point модели

### Ошибка 4: Смещение здания после поворота

**Причина**: Центральная точка здания не находится в центре сетки

**Решение**:
1. Настройте начало координат модели в 3D-программе
2. Или настройте позицию дочерних объектов в Unity
3. Убедитесь, что здание вращается вокруг центральной точки

### Ошибка 5: Файл слишком большой для загрузки

**Причина**: Слишком высокое разрешение текстур или слишком сложная модель

**Решение**:
1. Снизьте разрешение текстур (напр., с 4K до 2K)
2. Оптимизируйте модель для уменьшения количества полигонов
3. Сожмите текстуры (используйте формат DXT5 или BC7)
4. Удалите ненужные детали

## Продвинутые техники

### Добавление анимаций

Если ваше здание содержит анимации (например, вращающаяся мельница):
1. Создайте Animator Controller, перетащите анимацию в контроллер
2. Добавьте компонент Animator к префабу
3. Перетащите Animator Controller в компонент Animator

### Использование системы LOD

Прикрепите скрипты отсечения деталей для больших зданий:
1. Добавьте скрипт BuildingDetailCulling
2. Назначьте различные уровни LOD модели
3. Установите расстояния отсечения для разных уровней

### Пользовательские эффекты материалов

Используйте Shader Graph для создания специальных эффектов:
1. Создайте ассет Shader Graph
2. Спроектируйте пользовательские эффекты (например, светящиеся окна)
3. Создайте материал, использующий этот шейдер
4. Убедитесь, что шейдер совместим с целевой платформой
