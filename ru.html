🏗️ Руководство по созданию зданий для Мастерской

Версия: 1.0 | Последнее обновление: 2026 | Для: Unity 2022.3 LTS

Добро пожаловать, Создатель! Спасибо за интерес к созданию пользовательского контента! Ваш вклад поможет процветанию сообщества! Это руководство поможет вам создать контент, соответствующий спецификациям, и загрузить его в Мастерскую Steam.

Технические требования

Загрузка контента зависит от двух скриптов: BuildingData и WorkshopUploadTool. BuildingData используется для заполнения данных здания, а WorkshopUploadTool — для загрузки. Вам также понадобятся Steamworks.NET и файл steam_appid.txt. Файл steam_appid.txt должен быть помещен в корневой каталог проекта Unity с ID игры: 4490290. Если вам нужно обновить Steamworks.NET, вы можете скачать его со страницы GitHub:

https://github.com/rlabrecque/Steamworks.NET/releases

Импортируйте его в Unity и прикрепите скрипт Steam Manager к пустому GameObject.

💡 Подсказка: Я упаковал необходимые скрипты и Steamworks.NET в файл пакета Unity. Вы можете использовать этот файл пакета или скачать их отдельно.

Спецификации модели здания

⚠️ Напоминание об обязательных компонентах: Префабы зданий должны содержать следующие три основных компонента. Отсутствие любого из них приведёт к некорректной работе:
  • Mesh Filter: Отображает меш модели
  • Mesh Renderer: Рендерит модель (требуется назначение материала)
  • Collider (Box Collider): Используется для подсветки при наведении мыши, обнаружения кликов и определения коллизий при размещении

Поддерживаемые форматы ресурсов

  • Модели: FBX
  • Текстуры: PNG
  • Материалы: Unity URP Lit Shader

Ограничения размера файлов

  • Один AssetBundle здания: ≤ 50 МБ
  • Изображение предпросмотра: ≤ 1 МБ
  • Общий размер загрузки: ≤ 100 МБ

Руководство по дизайну зданий

1. Спецификации размера

Проектируйте в соответствии с размером сетки, занимаемым зданием. Размер основания здания (фундамента) должен быть минимум 1 метр в длину и ширину:

Для корректного расчёта занимаемой зданием сетки рекомендуется использовать нечётные квадраты для размера основания здания, например: 1×1, 3×3, 5×5. Чётные квадраты и неквадратные формы не рекомендуются, например: 1×2, 2×2, 2×3, 4×4.

Высота: Строгих ограничений нет, но рекомендуется не ниже 0,1 метра

2. Визуальный стиль

Для сохранения согласованности игры рекомендуется:

  • Использовать стиль LowPoly
  • Обеспечить общие пропорции и соотношения масштаба здания (справочный масштаб игры по умолчанию: высота персонажа 0,1 метра)
  • Использовать дизайн, дружественный к изометрическому виду
  • Избегать слишком мелких деталей (невидны издалека)
  • Использовать чёткие контуры и цветовой контраст
  • Учитывать детали с разных углов

3. Рекомендации по оптимизации производительности

  • Разумно контролировать количество полигонов
  • Прозрачные материалы (использовать Alpha Test вместо Alpha Blend)

Процесс создания

Шаг 1: Моделирование в 3D-программе

  1. Создавать модели с помощью Blender, Maya, 3ds Max и т.д.
  2. Убедиться, что начало координат модели находится в центре снизу
    • ✅ Правильно: Центральная точка здания в начале координат мира
    • ❌ Неправильно: Здание смещено или поворот не выровнен
  3. Применить все трансформации (Scale, Rotation)
  4. Экспортировать в формат FBX

Шаг 2: Импорт в Unity

  1. Перетащить FBX в проект Unity
  2. Установить Transform Position префаба здания на (0, 0, 0)
  3. Убедиться, что основание здания находится на плоскости Y=0
  4. Здание должно быть направлено в положительном направлении оси Z

Шаг 3: Настройка материалов и текстур

  1. Создать материалы (используя URP Lit Shader)
  2. Настроить параметры материала

Шаг 4: Добавление обязательных компонентов

Убедиться, что префаб здания содержит следующие три обязательных компонента:

1. Mesh Filter и Mesh Renderer

  1. Выбрать корневой объект здания
  2. Подтвердить, что компонент Mesh Filter добавлен (если нет, нажать Add Component → Mesh Filter)
  3. Подтвердить, что компонент Mesh Renderer добавлен
  4. Назначить материалы в Mesh Renderer (использовать Standard или URP Lit Shader)
⚠️ Важно: Здания без Mesh Renderer не будут отображаться и не будут вызывать подсветку при наведении мыши!

2. Collider

  1. Выбрать корневой объект здания
  2. Добавить компонент Box Collider (рекомендуется) или Mesh Collider
  3. Настроить размер Collider, чтобы покрыть всё здание
  4. Убедиться, что Collider не выходит за пределы здания, так как это вызовет столкновения с другими зданиями
⚠️ Важно: Collider является необходимым условием для подсветки при наведении мыши и обнаружения кликов! Здания без Collider не могут быть выбраны или взаимодействовать с игроками.

Шаг 5: Создание префаба

  1. Перетащить здание в окно Project для создания префаба
  2. Соглашение об именовании: {BuildingName}_Prefab
  3. Проверить, что префаб может быть нормально инстанцирован

Шаг 6: Создание BuildingData

  1. Правый клик → Create → City Builder → Building Data
  2. Заполнить следующие поля:
Поле Описание Пример
Building Name Отображаемое название здания, можно использовать имя или номер Ferriswheel, SSmbl2
Building ID Уникальный идентификатор, рекомендуется: название стиля + номер, или чистый номер SunsetCity_01, SSmbl2
Style ⭐ Определяет, на какой вкладке стиля появляется здание в меню строительства игры, рекомендуется: имя разработчика, название пакета или название темы. Здания одной серии должны иметь одно и то же значение Lyon'sPack, SunsetCity
Category ⭐ Определяет, в каком столбце категории оно появляется под этим стилем, можно использовать существующие категории игры для совместимости, или заполнить в соответствии с классификацией вашего здания Large, Residential
Grid Size Размер занимаемой зданием сетки 3x3
Building Prefab Перетащить ваш префаб здания —

Справочник встроенных категорий игры:

Значение категории Описание
LargeБольшой
MediumСредний
SmallМаленький
⚠️ Поля Style и Category очень важны: Они будут упакованы в metadata.json (автоматически записывается инструментом загрузки). Когда игра загружает ваш мод, она определяет положение здания в меню на основе этих двух полей. Обязательно заполните их, иначе здание может не отображаться в правильной категории.
  1. Создать иконку здания (рекомендуется 128×128 PNG) и перетащить её в поле Icon

Шаг 7: Тестирование здания

  1. Протестировать здание в игре:
    • Может ли оно быть правильно размещено
    • Нормально ли вращение
    • Правильно ли обнаружение столкновений
    • Удовлетворительны ли визуальные эффекты
  2. Настраивать до удовлетворения

Шаг 8: Подготовка к загрузке

  • Создать изображение предпросмотра (рекомендуется 600x600)
  • Подтвердить, что Style и Category в BuildingData правильно заполнены (инструмент загрузки читает их автоматически)

Контрольный список перед загрузкой:

  • Количество треугольников модели в пределах ограничений
  • Разрешение текстуры разумно
  • Содержит компонент Mesh Filter
  • Содержит компонент Mesh Renderer (материал назначен)
  • Содержит компонент Collider
  • Префаб может быть нормально инстанцирован
  • Здание выровнено по сетке
  • Нет отсутствующих материалов или текстур
  • Все поля BuildingData заполнены
  • Проверить, создан ли файл steam_appid.txt в корневом каталоге проекта
  • Изображение предпросмотра чёткое
  • Тест в игре пройден

Шаг 9: Загрузка в Мастерскую

⚠️ Загрузка выполняется в два этапа, так как упаковка AssetBundle и загрузка в Steam имеют разные требования к состоянию редактора и должны выполняться отдельно.

Шаг 1: Упаковка ресурсов (Режим не-Play)

  1. Убедиться, что вы находитесь в Режиме не-Play (редактор не запущен)
  2. Открыть City Builder → Workshop Upload Tool
  3. Выбрать тип мода: Building
  4. Перетащить BuildingData (инструмент автоматически читает Style и Category, можно изменить вручную)
  5. Перетащить префаб здания
  6. Заполнить информацию о мастерской (в основном название, описание можно редактировать в Мастерской)
  7. Нажать "Шаг 1: Упаковать ресурсы как AssetBundle"
  8. Когда вы увидите сообщение "Упаковка завершена! Пожалуйста, войдите в Режим Play и нажмите Шаг 2 для загрузки", это успех
💡 Подсказка: Путь упаковки будет автоматически сохранён и не потеряется после входа в Режим Play, нет необходимости заполнять информацию повторно.

Шаг 2: Загрузка в Steam (Режим Play)

  1. Нажать кнопку Play редактора Unity и дождаться инициализации SteamManager
  2. В окне инструмента подтвердить, что Steam инициализирован (без предупреждений)
  3. Нажать "Шаг 2: Загрузить в Мастерскую Steam"
  4. Дождаться завершения загрузки и записать отображённый ID элемента Мастерской (для последующих обновлений)
💡 Подсказка: Инструмент загрузки автоматически генерирует metadata.json и упаковывает его в каталог содержимого AssetBundle, который содержит информацию о Style и Category. После того как игроки подпишутся, игра автоматически отобразит его в правильной категории.

Шаг 10: Публикация и поддержка

  1. Завершить информацию на странице Мастерской Steam
  2. Добавить больше скриншотов

Примечания о материалах неба и земли

  • Шейдеры неба и земли должны использовать конвейер URP
  • Шейдер земли должен использовать процедурные нормали для обработки динамических настроек размера области сетки

Распространённые ошибки и решения

Ошибка 0: "Building AssetBundles while in play mode is not allowed" при упаковке

Причина: Кнопка упаковки была нажата в Режиме Play, но упаковка AssetBundle может выполняться только в Режиме не-Play

Решение:

  1. Выйти из Режима Play (нажать кнопку остановки)
  2. Снова нажать "Шаг 1: Упаковать ресурсы как AssetBundle" в инструменте
  3. Войти в Режим Play для загрузки после завершения упаковки

Ошибка 1: Здание отображается розовым

Причина: Отсутствует материал или шейдер

Решение:

  1. Убедиться, что все текстуры правильно назначены
  2. Проверить, включены ли материалы в AssetBundle

Ошибка 2: Здание не может быть размещено

Причина: Отсутствуют обязательные компоненты (Mesh Filter/Mesh Renderer/Collider) или неправильная настройка размера сетки

Решение:

  1. Подтвердить, что префаб здания содержит компоненты Mesh Filter, Mesh Renderer и Collider
  2. Проверить, назначены ли материалы в Mesh Renderer
  3. Добавить Box Collider к корневому объекту здания
  4. Проверить настройку Grid Size в BuildingData
  5. Подтвердить размеры здания

Ошибка 3: Положение здания смещено

Причина: Неправильные настройки Transform префаба

Решение:

  1. Установить Position префаба на (0, 0, 0)
  2. Убедиться, что основание здания находится на плоскости Y=0
  3. Проверить положение Pivot Point модели

Ошибка 4: Здание неправильно выровнено после поворота

Причина: Центральная точка здания не в центре сетки

Решение:

  1. Настроить начало координат модели в 3D-программе
  2. Или настроить положение дочерних объектов в Unity
  3. Убедиться, что здание вращается вокруг центральной точки

Ошибка 5: Файл слишком большой для загрузки

Причина: Слишком высокое разрешение текстуры или слишком сложная модель

Решение:

  1. Снизить разрешение текстуры (например, с 4K до 2K)
  2. Оптимизировать модель для уменьшения количества полигонов
  3. Сжать текстуры (использовать формат DXT5 или BC7)
  4. Удалить ненужные детали

Продвинутые техники

Добавление анимации

Если ваше здание содержит анимацию (например, вращающаяся ветряная мельница):

  1. Создать контроллер анимации и перетащить анимацию в него
  2. Добавить компонент Animator к префабу
  3. Перетащить контроллер анимации в компонент Animator

Использование системы LOD

Добавить скрипт отсечения деталей для больших зданий:

  1. Добавить скрипт BuildingDetailCulling
  2. Назначить разные уровни LOD модели
  3. Установить разные дистанции отсечения для каждого уровня

Пользовательские эффекты материалов

Использовать Shader Graph для создания специальных эффектов:

  1. Создать ассет Shader Graph
  2. Разработать пользовательские эффекты (например, светящиеся окна)
  3. Создать материал, используя этот шейдер
  4. Убедиться, что шейдер совместим с целевой платформой
